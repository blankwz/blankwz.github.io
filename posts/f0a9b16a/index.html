开源个人网盘 - 对象存储型

一、zfile

介绍

一个在线文件目录的程序, 支持各种对象存储和本地存储, 使用定位是个人放常用工具下载, 或做公共的文件库. 不会向多账户方向开发.

使用JAVA开发：前端基于 h5ai 的原有功能使用 Vue 重新开发了一遍. 后端采用 SpringBoot, 数据库采用内嵌数据库.

Github：地址

文档：地址 (更多帮助看这里)

预览：地址

系统特色

  • 支持Windows和Linux
  • 内存缓存 (免安装)
  • 内存数据库 (免安装)
  • 个性化配置
  • 自定义目录的 header 说明文件
  • 自定义 JS, CSS
  • 文件夹密码
  • 支持在线浏览文本文件, 视频, 图片, 音乐. (支持 FLV 和 HLS)
  • 文件/目录二维码
  • 缓存动态开启, 缓存自动刷新
  • 全局搜索
  • 支持 阿里云 OSS, FTP, 华为云 OBS, 本地存储, MINIO, OneDrive 国际/家庭/个人版, OneDrive 世纪互联版, 七牛云 KODO, 腾讯云 COS, 又拍云 USS.

二、ZPan

试用了下，还很年轻，期待完善！

1 介绍

ZPan致力于打造一款不限速的网盘系统，因此我们采用客户端直连云存储的方式进行设计。

目前ZPan支持所有兼容S3协议的云存储平台，您可以选用您熟悉的平台来驱动ZPan。

Github：地址

文档：地址

使用GO开发

特色

  • 完全不受服务器带宽限制
  • 支持所有兼容S3协议的云存储
  • 支持文件及文件夹管理
  • 支持文件及文件夹分享（未登录可访问）
  • 支持文档预览及音视频播放
  • 支持多用户存储空间控制
  • 支持多语言

2 安装

1. 申请 存储桶

以腾讯COS为例

  1. 对象存储COS - 存储桶列表：地址

  2. 创建存储桶

    1. 名称随意
    2. 地域随意
    3. 访问权限 - 共有读私有写

  3. 创建API密钥：地址

    • 新建密钥
    • 查看，并复制 SecretId 和 SecretKey，保存到其他地方，以便后面使用

  4. 跨域访问CORS设置

    1. 桶列表 - 点击桶名称 ，进入你的桶
    2. 安全管理 - 跨域访问CORS设置 - 添加规则
      1. 来源 Origin：输入你的域名，也可以输入 *
      2. 操作 Methods： PUT
      3. Allow-Headers：content-type,content-disposition,x-amz-acl
      4. Expose-Headers：空
      5. 超时 Max-Age：随意

  5. 权限管理：

    • 存储桶访问权限：确保你的账户有权限，如果你申请了子账户，需要添加进来

2. 安装ZPan服务

以服务方式为例，Docker方式没试，详解官方文档

3. 配置yml文件

  1. 服务器中输入：vi /etc/zpan/zpan.yml

  1. 启动服务：systemctl start zpan

  2. 查看服务状态：systemctl status zpan

  3. 设置开机启动：systemctl enable zpan

  4. 后面如果修改了yml，需要重启服务：systemctl restart zpan

4. 服务器绑定域名

  1. 域名解析到服务器IP
  2. 若是用的8222端口，还想用80端口访问，则需要用Nginx做反向代理，推荐用宝塔做反向，很方便。

3. 登陆

  1. 输入你的域名+端口，比如 http://ww.kao.lol
  2. 查看账户和密码，服务器中输入：journalctl -u zpan
  3. 如何修改账户和密码
    • 进入数据库中修改
    • 邮箱随意修改
    • 账户不知道是什么加密，不管，就用邮箱登陆好了
    • 密码：为MD5加密，随便找个MD5加密工具 输入密码，加密，替换数据库中密码即可

4. 测试

  • 进入后，随便传个问题，是否可用
  • 如果没反应，通过 浏览器 F12 调试，查看错误原因

5. 问题

我这边的问题原因是：用的反向代理，导致跨域CORS设置问题

再次跨域访问CORS设置

  1. 桶列表 - 点击桶名称 ，进入你的桶

  2. 安全管理 - 跨域访问CORS设置 - 修改刚刚的规则

    1. 来源 Origin：输入 *
    2. 操作 Methods： PUT
    3. Allow-Headers：content-type,content-disposition,x-amz-acl
    4. Expose-Headers：空
    5. 超时 Max-Age：随意

  3. 再次测试，发现可以上传了！说明跨域有问题

    • 想到我用的反向代理，估计就是这个原因了。
