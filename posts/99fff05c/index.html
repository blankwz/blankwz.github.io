本站点Hexo配置

一、主题 - Butterfly

第一次建Hexo站，感谢作者！教程文档写的很清楚！让新手少走很多弯路

  • 官方Github
  • 作者站点
  • 文档

二、目前问题：

代码高亮没用！！Python！

三、安装流程：（主题自带）

1. 安装Dev分支

  • git clone -b dev https://github.com/jerryc127/hexo-theme-butterfly.git themes/Butterfly

2. 安装pug 以及 stylus 的渲染器

  • npm install hexo-renderer-pug hexo-renderer-stylus --save

3. 字数统计插件

  • npm install hexo-wordcount --save

4. 文章置顶插件

  • hexo-generator-indexed

5. 改默认主题文件

在source/_data/创建butterfly.yml主题文件，并根据自己的需求更改（参照教程文档）

6. 根据需求创建新页面：

  • hexo new page 页面名

7. 根据需求更改导航菜单

8. 评论插件:

  • 我选择了Gitalk
  • 主题自带，不用安装，直接配置
    • 需要 GitHub Application，如果没有 点击这里申请
    • 然後配置butterfly.yml:
      • 参考：gitalk用法、gitalk官方说明

四、自己增加的其他插件

1.文章加密插件

hexo-blog-encrypt

  • npm install --save hexo-blog-encrypt

详见： Hexo - 初玩加密插件

2.中文链接优化插件，固定链接

hexo-abbrlink

  • Github
  • 安装 : npm install hexo-abbrlink --save
  • 修改根目录的config.yml
  • 参考

3.文章置顶插件

  • hexo-generator-indexed

  • Github说明： hexo-generator-indexed

  • 安装：

  • 文字置顶方法：在文章的 Front-matter 中增加一个 sticky 参数用来置顶，

    • 其值应为大于0的整数，表示置顶的优先级（未指定则默认为 0）。
    • 数字越大，文章越靠前

  • 置顶文章隐藏方法：在文章的 Front-matter 中增加一个 hide 参数用来隐藏

  • 隐藏特定分类的文章：在_config.yml 中可以通过 hide_categories 选项设置隐藏某个分类下的文章

五、出现过的问题

1、出现 YAMLException: can not read a block mapping entry; a multiline key may not be an implicit key at line

看样子像是头部声明文件，少了空格或者符号写错。
但是！我这个不是！
因为标题中含有 [] 符号！ 去掉后，完美解决！

2、Hexo更改默认图片文件路径（把默认的图片存放文件夹放在根目录的source中）

都知道主题中的默认图片都是存放在主题的 Butterfly\source\img 文件夹中，在butterfly.yml中更改路径即可。

  • 但是因为主题的整个文件夹，是单独git下来的，当push整个Hexo站点时，这个文件夹是不会push上去的。
  • 如果添加子模块，又太麻烦~
  • 能不能把默认的图片存放文件夹放在主题外呢

A：

  • 可以
  • 在根目录的source文件夹中创建个目录。比如pic 文件夹
    • 注意，文件夹的命名！
      • 区分大小写
      • 不能以 _ 下划线开头
  • 再在主题的配置yml文件夹中，更改路径即可
    • 比如之前 index.jpg 存放在 根目录\themes\Butterfly\source\img中，配置文件路径写法是 /img/index.jpg
    • 现在 index.jpg存放在 根目录\source\pic中，配置文件路径写法是 /pic/index.jpg

3、标题用 []符号后，出现 can not read a block mapping entry; a multiline key may not be an implicit key at line 2, column 10

A: 不要用[]符号在 头文件的 title中

4、升级后，主页中的导航栏就看不到了，但是鼠标放上去会显示超级链接”手”的标志。还有点击出文字效果ClickShowText也没用了

A：升级后的主题Config.yml文件有变动，把新的Config.yml内容，拷贝到butterfly.yml中,然后重新配置下你之前改动过的站点的信息即可。

5、错误 Cannot read property ‘map’ of undefined

A: 我按tag这个来找，找出问题了，是我在“source\待完善”的文件夹下（这个文件夹我是存放临时文件用的，不发布），如果有md文件有tag标签，就会报这样的错误。删掉tag标签内容，就正常了（我猜，可能是非post文件夹的其他文件夹，如果有tag标签，就会报错）我的旧版主题没有这样的问题不知道这算不算一个BUG感谢！
把“source\待完善”的文件夹 移除 ~移到其他地方去

6、装了置顶插件，文章也显示了置顶两个字，但是并没有置顶

A:

1.先看看博客文件夹的node_modules文件夹中，是否有多个置顶插件

比如 这三个，不要同时出现，如果有，则删除，只留一个

  • hexo-generator-index
    • 官方插件，删除方法 npm uninstall hexo-generator-index --save
  • hexo-generator-indexed
    • npm uninstall hexo-generator-indexed --save
  • hexo-generator-index-pin-top
    • npm uninstall hexo-generator-index-pin-top --save
2.再看看_config.yml中是否配置是否正确

7、本地可以预览，但是传到Github上显示404错误！！！

错误原因1： 因为上面的 置顶插件，设置了_config.yml

错误原因2：缺失关键插件

配置文件中的插件与本地nmp库（node_modules文件夹）中的插件配不上。

因为我是两台电脑（公司、家）同步使用，安装插件时可能出现混乱，装错了关键插件，所以造成同步时，出现错误。

解决办法：看看博客中的node_modules文件夹，是否有插件缺失~~ 这个得根据经验，排除法。
