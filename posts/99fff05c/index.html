安装流程：

  1. 安装Dev分支 - git clone -b dev https://github.com/jerryc127/hexo-theme-butterfly.git themes/Butterfly
  2. 安装pug 以及 stylus 的渲染器 - npm install hexo-renderer-pug hexo-renderer-stylus --save
  3. 字数统计插件 - npm install hexo-wordcount --save
  4. 文章置顶插件 - npm uninstall hexo-generator-index --save
  5. 在source/_data/创建butterfly.yml主题文件，并根据自己的需求更改（参照教程文档）
  6. 根据需求创建新页面：
    • hexo new page 页面名
  7. 根据需求更改导航菜单

出现过的问题

Hexo更改默认图片文件路径（把默认的图片存放文件夹放在根目录的source中）

都知道主题中的默认图片都是存放在主题的 Butterfly\source\img 文件夹中，在butterfly.yml中更改路径即可。

  • 但是因为主题的整个文件夹，是单独git下来的，当push整个Hexo站点时，这个文件夹是不会push上去的。
  • 如果添加子模块，又太麻烦~
  • 能不能把默认的图片存放文件夹放在主题外呢

A：

  • 可以
  • 在根目录的source文件夹中创建个目录。比如pic 文件夹
    • 注意，文件夹的命名！
      • 区分大小写
      • 不能以 _ 下划线开头
  • 再在主题的配置yml文件夹中，更改路径即可
    • 比如之前 index.jpg 存放在 根目录\themes\Butterfly\source\img中，配置文件路径写法是 /img/index.jpg
    • 现在 index.jpg存放在 根目录\source\pic中，配置文件路径写法是 /pic/index.jpg

标题用 []符号后，出现 can not read a block mapping entry; a multiline key may not be an implicit key at line 2, column 10

A: 不要用[]符号

升级后，主页中的导航栏就看不到了，但是鼠标放上去会显示超级链接”手”的标志。还有点击出文字效果ClickShowText也没用了

A：升级后的主题Config.yml文件有变动，把新的Config.yml内容，拷贝到butterfly.yml中,然后重新配置下你之前改动过的站点的信息即可。
