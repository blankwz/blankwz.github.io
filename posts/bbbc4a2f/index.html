Python基础之安装与部署

步骤
1.安装

2.运行

3.国内镜像

4.虚拟环境

5.包管理 Anaconda

一、安装

从2020年1月1日起Python2已经停止维护了，为了将来，还是不要用Python2了

关于Python2和Python3的版本：

  1. 两者不兼容
  2. 很多基本的函数接口变了，甚至是，有些库或函数被去掉了，改名了
  3. Python2可能就参考以前别人写的代码需要用到
  4. 其他的区别 ，自行搜索

1.Windows平台安装Python

  1. Windows安装很简单，官网下载安装包，一直下一步即可

  2. 注意事项：（关于环境变量，请看 环境变量 教程）

  1. 检测是否安装成功
    • CMD下输入 Python3 -v 和 Pip -V

2.Linux平台安装Python

有些版本的Linux系统会自动Python2，请忽略，然后安装Python3

  1. 下载依赖环境

  1. 官网下载合适的版本源码包

    • 或者通过 wget 等下载命令下载
      • 安装wget：yum -y install wget
      • wget https://www.python.org/ftp/python/3.x.x/Python-3.x.x.tgz

  2. 解压压缩包：tar -zxvf Python-3.x.x.tgz (请自行更改版本号3.x.x或者包名)

  3. 如果你需要自定义一些选项修改Modules/Setup

  4. 进入解压后的文件夹： cd 文件夹名称

  5. 释放编译文件 Makefile， 执行命令

    • ./configure --prefix=/opt/python38 # 指定安装目录为/opt/python36

  6. 编译，把源码包里面的代码编译成linux服务器可以识别的代码

    • make

  7. 编译安装,此步才会最终生成 /opt/python38/

    • make install

Linux环境变量

  1. 修改/etc/profile系统环境变量配置文件

    • vim /etc/profile # 系统环境变量配置文件
    • 或者 vim ~/.bash_profile # 用户环境变量配置文件

  2. 在PATH=后加入python的安装路径的bin文件夹

    • PATH=/opt/python36/bin
    • 需放在Path的第一位

  3. 建立Python3和pip3的软链到 usr/bin/python3

    • ln -s /opt/python38/bin/python3 /usr/bin/python3
    • ln -s /opt/python38/python3/bin/pip3 /usr/bin/pip3

  4. 生效（重载）配置文件

    • source /etc/profile
    • 查看是否环境变量生效 echo $PATH

  5. 检查是否安装完成

    • 输入：Python3 -v
    • 输入：pip -v

二、运行Python

Python有多种运行方式

  • Python自带的 交互式解释器
  • 命令行脚本，直接 终端中 运行 .py文件
  • 集成开发环境（或者编辑器）

1. 交互式解释器 运行

我个人不怎么用 ，简单讲下

就是在终端或者命令行中输入 ： Python，即进入 解释器中，在这编写 Python 代码

  • 简单语法：直接输入： 300+300 回车试试
  • 退出命令： exit（）或者 Ctrl+Z

2. 命令行脚本

在你的应用程序中通过引入解释器可以在命令行中执行Python脚本

  • python XXX.py

3. 集成开发环境

多种选择

  1. Python自带 IDLE

  2. PyCharm 推荐！

  3. Vscode 编辑器

  4. jupyter notebook

三、不同版本共存问题

1. 更改pip和python

共存问题在于使用Python.exe和pip.exe需要区分开来

  1. 分别更改两个版本的Python的根目录下的 python.exe和 pythonw.exe名称
    • 比如：python37.exe和 pythonw37.exe
    • 或者 python38.exe和 pythonw38.exe
    • 可以只更改一个，另外个可以不需要改
  2. 分别更改两个版本的Python\Scripts下的 pip.exe 名称
    • 比如：pip37.exe

2. 报错问题

问题，执行pip报错：Fatal error in launcher: Unable to create process using '"c:\python379\python.exe" "C:\python379\Scripts\pip37.exe" install pyHook?1.5.1?cp37?cp37m?win_amd64.whl': ???????????

  • 原因：因为改了根目录的 python.exe 名称，所以，pip无法找到
  • 解决：把pip删掉，然后重新下载pip （下面python37为你自己改的名称）
    • python37 -m pip install --upgrade pip --force-reinstall

3. 复制包到另外个版本里

举例：

  1. 3.7中 导出所有包和依赖版本 pip freeze > reqs.txt
  2. 3.8中 安装包 pip38 install -r reqs.txt

四、安装whl包及失败解决

安装

  1. 输入：python 查看本机平台的版本

  2. 到 http://www.lfd.uci.edu/~gohlke/pythonlibs/找whl包

  3. 找对应版本，下载

    • cp27，就是python2.7
    • cp34，就是python3.4
    • 以此类推…
    • win32是32位系统，amd64是64位系统

  4. 安装wheel pip install wheel

  5. 安装包：pip install whl包名称（需要cd到包下载的路径）

问题

ERROR: pyHook‑1.5.1‑cp37‑cp37m‑win_amd64.whl is not a valid wheel filename.

没有输错，为什么？这个问题很莫名…!

总结：下载包后，不要改名！不要改名！不要改名！

改了名，就无法安装！

把名称再改回来！依然无法安装！

重新下载！

好了！
