3.清空表后的方法

方法一：通过truncate table 删除表，并重置标识值

truncate table 表名 – 删除表中的所有行，而不记录单个行删除操作

  • 这样不但将数据删除，而且可以重新置位identity属性的字段。

  • TRUNCATE TABLE 在功能上与不带 WHERE 子句的 DELETE 语句相同：二者均删除表中的全部行。

    • 但 TRUNCATE TABLE 比 DELETE 速度快，且使用的系统和事务日志资源少

详见下方的 [DBCC CHECKIDENT方法](#DBCC CHECKIDENT` 方法)

二、删除表中数据（行）

1. TRUNCATE TABLE与DELETE的区别

删除表的两种方法：

  • TRUNCATE TABLE
  • DELETE

TRUNCATE TABLE 在功能上与不带 WHERE 子句的 DELETE 语句相同：二者均删除表中的全部行。

但 TRUNCATE TABLE 比 DELETE 速度快，且使用的系统和事务日志资源少。

  • DELETE 语句每次删除一行，并在事务日志中为所删除的每行记录一项。

  • TRUNCATE TABLE 通过释放存储表数据所用的数据页来删除数据，并且只在事务日志中记录页的释放。

标识值

  • TRUNCATE TABLE 删除表中的所有行，但表结构及其列、约束、索引等保持不变。新行标识所用的计数值重置为该列的种子。

  • 如果想保留标识计数值，请改用 DELETE。

  • 如果要删除表定义及其数据，请使用 DROP TABLE 语句。

  • 对于由 FOREIGN KEY 约束引用的表，不能使用 TRUNCATE TABLE，而应使用不带 WHERE 子句的 DELETE 语句。

    • 由于 TRUNCATE TABLE 不记录在日志中，所以它不能激活触发器

  • TRUNCATE TABLE 不能用于参与了索引视图的表

2. DBCC CHECKIDENT 方法

DBCC CHECKIDENT 检查指定表的当前标识值，如有必要，还对标识值进行更正

语法

详情

对当前标识值所做的具体更正取决于参数规范。

  • DBCC CHECKIDENT ('table_name', NORESEED) 不重置当前标识值。DBCC CHECKIDENT 返回一个报表，它指明当前标识值和应有的标识值。
  • DBCC CHECKIDENT ('table_name') 或 DBCC CHECKIDENT ('table_name', RESEED) 如果表的当前标识值小于列中存储的最大标识值，则使用标识列中的最大值对其进行重置。
  • DBCC CHECKIDENT ('table_name', RESEED, new_reseed_value) 当前值设置为 new_reseed_value。
    • 如果自创建表后没有将行插入该表，则在执行 DBCC CHECKIDENT 后插入的第一行将使用 new_reseed_value 作为标识。
    • 否则，下一个插入的行将使用 new_reseed_value + 1。
    • 如果 new_reseed_value 的值小于标识列中的最大值，以后引用该表时将产生 2627 号错误信息。

九、问题

1.已有数据表中插入非空字段

Q：错误提示：ALTER TABLE 只允许添加满足下述条件的列: 列可以包含 Null 值；或者列具有指定的 DEFAULT 定义；或者要添加的列是标识列或时间戳列；或者，如果前几个条件均未满足，则表必须为空以允

A：该表里已经有数据了，所以再添加字段时，已经有的数据该字段应该是空，但是我们添加字段时设定的是非空，所以就报错了。

解决办法:

  1. 先将该列设为允许为空，进行添加，
  2. 然后将已经有的该数据更新成具体的非空值，
  3. 然后再修改该列的属性为非空，这样，再插入非空的值就行了。
