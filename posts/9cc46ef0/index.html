炉石传说

一、HDT插件

NGA：地址

Github：地址

坟场插件：地址

介绍

HDT全称是Hearthstone Deck Tracker，这个工具采集到的数据会汇总到[https://hsreplay.net/]这个网站，可以查询到玩家所需详细的、各种各样卡组数据。这个网站制作精良，界面简洁实用，可读性强，即便不是炉石内容，作为一个数据查询网站来看待，也是相当的精致。HDT的背后是一个8人的创业团队，公司名叫HearthSim。

HDT早已经迭代过很多很多轮，已经渡过了早期比较难用的阶段。目前HDT支持简体中文，网站中国大陆可以直连，几乎可以说是炉石玩家的最佳选择。在国外市场，HDT占据了市场上的大部分份额，绝对优势。在国内则由于没有做过推广，所以玩家不太多，但是也有很多专业玩家在使用。

介绍完了背景之后，最后我来回答一下一开始抛出的那个问题，为什么要使用HDT？
优点：
1、界面简洁，清爽，无广告。
2、占用内存小。
3、更新迅速但并不会经常迭代，几乎和每次的版本变动无缝衔接，但对用户的打扰极少。
4、功能多样，有一定的界面自定义空间。
5、数据分析功能强大，卡组界面直接可以查看大数据，还可以直连hsreplay网站。
6、支持多种卡组的导出/导入方式，界面清爽，是卡组搬运者们的最佳工具。

中文设置

打开Option选项->Appearance外观->Language语言->Chinese simplifier简体中文

另外，把左下角的高级选项打开

二、整活断网

1 说明

参考：NGA - 我写了一个断线重连的脚本

原理：

通过防火墙出站规则，脚本快速禁止炉石通网络，再恢复网络

运行 炉石断线.bat 脚本需要两个条件：

  1. 防火墙开启
  2. 管理员权限

心跳测试

在软件通讯的时候，有一个东西叫做“心跳测试”，大意如下：
A每过一段时间会向B发送一个东西：“你活着没？”，B会回复：“我活着” \ “我挂了” \ 没有回复，根据B的不同回复，A会做不同的反应。而每隔多久进行一次心跳测试，则因软件的不同而不同。
炉石传说也有这么一个东西，所以我们会有 “游戏已中断，正在重连” 或者 “您已断开游戏连接” 的提示，这就是炉石客户端在做心跳测试的时候，发现连接不到对方而反馈给玩家的东西。
至于炉石心跳测试的时间是多久一次，我不清楚，所以我只是简单的把断线时间设置为比较安全的3秒，如果有人得出了更合适更科学的断线时间设置，欢迎指出。

综上所述，3秒断线并不不是一定会在炉石中体现出来，有时候你甚至不知道自己断线3秒，因为炉石的心跳测试没有检测到这个问题。
而有时候你断线脚本连续点的太频繁，会触发断线连接不上的机制，导致你彻底断开和本场游戏的连接。整活有风险，断线需谨慎。

个人经验，把断线时间设置成4-5秒时可以保证更高的断线成功率。鉴于整活的人一般都火急火燎，一秒钟都不想多等，所以源码中的这个时间我并没有修改。

关于断线后选择随从的剩余时间

断线重连的时间上限 = 基本时间 + 最长动画时间
比如第一轮 基本时间 = 30秒， 最长动画时间 = 5秒， 那么你断线重连的时间上限就是35秒。
如果第十轮 基本时间 = 90秒， 最长动画时间 = 55秒，那么你断线重连的时间上限就是145秒。
而其他人则有的早点出来，有的晚点出来，哪怕是动画时间最长的人，也有基本时间做保障。

2 方法：

  1. 控制面板 - 网络

  2. Windows Defender防火墙 - 高级设置

  3. 点击出站规则>>新建规则>>程序>>选择炉石程序路径的执行文件

  4. 下一步>>选择阻止连接>>下一步>>命名为：zhenghuo>> 完成 (名称随意)

  5. 制作bat脚本 （新建txt文本 - 另存为 编码为ANSI，保存为 zhenghuo.bat文件）

  6. 管理员模式运行它，即可

  7. 建议：回合结束前3、4秒时双击脚本

3 技巧：

如何使用管理员权限？

先建立当前脚本的快捷方式，然后右击-属性-快捷方式-高级，选择“用管理员身份运行”，以后只运行快捷方式即可

将脚本快捷方式放到开始菜单栏 …

先建立一个脚本的快捷方式，然后将它放到你的用户的开始文件夹下，我的路径是在里：

  • C:\Users\Administrator\AppData\Roaming\Microsoft\Windows\Start Menu\Programs

  • 或者 C:\ProgramData\Microsoft\Windows\Start Menu\Programs

然后打开开始菜单，点击左边的“所有应用”，找到那个快捷方式，右击选择“固定到开始屏幕”。

4 脚本

1. 手动设置路径

1、判断是否存在防火墙规则，如果存在，则进行下一步；如果不存在，则创建规则
2、更新程序路径并启用防火墙规则(开始断线)
3、等待X秒
4、禁用防火墙规则(恢复网络)

2. 自动更新路径

1、根据炉石进程获取到执行程序的路径(执行效率低，约150毫秒)
2、判断获取的炉石路径是否为空，如果为空则报错“无法获取[炉石传说]路径，即将退出…”。因为DOS不同，win7下可能失败，如果是这里报错，可以使用断线脚本1，手动设置
3、判断是否存在防火墙规则，如果存在，则进行下一步；如果不存在，则创建规则
4、更新程序路径并启用防火墙规则(开始断线)
5、等待X秒
6、禁用防火墙规则(恢复网络)

3. 断线之前加入死亡判断_手动设置路径

测试版，不建议初学者使用此脚本。
由于判断错综复杂，以及DOS版本的不同，可能存在BUG。
代码思路：
1、获取炉石日志的文件修改日期(win7下我找不到战斗日志文件，如有大佬可以指出)
2、判断炉石最后修改日期：如果是昨天和今天，则搜索日志(第3步)；否则直接断线(第5步)
3、如果日期符合条件，搜索日志中的死亡记录：如果存在，取最后一条的时间点进行判断(第4步)；如果不存在，则直接断线(第5步)
4、判断最近一笔日志和你的时间差(DOS没有时间判断函数，这里属于简单判断，所以会有判断错的时候，但是出错的概率不大)
如果日志和系统时间相差xxx秒之内，提示你已死亡，是否继续断线；如果时间相差xxx秒之外，则直接断线(第5步)
5、判断是否存在防火墙规则，如果存在，则进行下一步；如果不存在，则创建规则
6、更新程序路径并启用防火墙规则(开始断线)
7、等待X秒
8、禁用防火墙规则(恢复网络)

自定义方法：
文件头几行已经标明
如果已经存为bat格式，则右键点击“编辑”来修改
默认死亡判断时间为40秒，即判断系统时间前40秒的死亡记录为你已死亡。如果你设置为3600，那么1小时之内的死亡记录都会提示你已经死亡。
