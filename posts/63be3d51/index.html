    Linux简单命令上手

    一、查看帮助方法

    1. command -h

    • useradd -h # 查看useradd的帮助文件

    2. man command

    • 通过man查看更详细的帮助

    3. 其他第三方

    tldr

    安装：pip install tldr

    示例 ：tldr tar，打印出 常用的tar命令用法

    二、用户管理

    1. 命令

    1.1 创建用户：useradd username

    • 创建一个名为gxkon的用户：useradd gxkon

    1.2 更改用户密码：passwd username

    • 更改指定用户密码：passwd gxkon
    • 更改当前用户密码：passwd

    2. 用户配置文件

    • 存放用户设置的文件：/etc/passwd

    • 存放用户组设置的文件：/etc/group

    2.1 cat查看用户配置文件

    通过cat命令将文件里面的内容输出在命令行上：

    2.2 用户的主目录

    主目录就是用户登录进去后默认的路径

    上面的/root 和 /home/gxkon就是用户的主目录

    2.3 /bin/bash

    /bin/bash 的位置是用于配置登录后的默认交互命令行的

    三、文件管理

    1.切换目录cd (change directory)

    • cd . 表示切换到当前目录；

    • cd .. 表示切换到上一级目录；

    2.列出文件 ls （list）

    • ls 列出当前目录下的文件

    • ls -l 用列表方式（详情） 列出文件

    • ls -a 显示所有文件，包括隐藏文件

    3.文件信息讲解

    1. 第一个字段的第一个字符是文件类型

      • 如果是“-”，表示普通文件
      • 如果是 d，就表示目录
      • 等等

    2. 第一个字段剩下的 9 个字符是模式，就是权限位（access permission bits）

      • 3 个一组，每一组 rwx 表示读（read）、写（write）、执行（execute）
      • 如果是字母，就说明有这个权限
      • 如果是横线，就是没有这个权限
      • 这三组分别表示文件所属的用户权限、文件所属的组权限以及其他用户的权限

    3. 第二个字段是硬链接（hard link）数目

    4. 第三个字段是所属用户

    5. 第四个字段是所属组

    6. 第五个字段是文件的大小

    7. 第六个字段是文件被修改的日期

    8. 最后是文件名

    4.改变文件权限

    • chmod 改变用户权限

    • chown改变所属用户

    • chgrp改变所属组。

    5.搜索文件grep和管道 |

    • CentOS ：rpm -qa | grep filename

    • Ubuntu： dpkg -l | grep filename

    举例：rpm -qa | grep jdk，这个命令是将列出来的所有软件形成一个输出。

    | 是管道，用于连接两个程序，

    前面 rpm -qa 的输出就放进管道里面，

    然后作为 grep 的输入，grep 将在里面进行搜索带关键词 jdk 的行，并且输出出来。

    grep 支持正则表达式，因此搜索的时候很灵活，再加上管道，这是一个很常用的模式

    6.编辑文件

    工具VIM （必学），类似笔记本

    如果默认没有安装，可以通过 yum install vim 及 apt-get install vim 进行安装

    • i 进入编辑模式
    • esc 退出编辑模式
    • :w 保存 （write）
    • :wq 保存 并退出
    • :q 退出 （quit）
    • :q! 不保存并退出

    四、安装软件

    1.两种后缀类型安装文件 rpm 或者 deb

    CentOS 体系 用 rpm

    Ubuntu 体系 用 deb

    2.安装卸载

    安装rpm包：rpm -i 包名.rpm

    安装deb包：dpkg -i 包名.deb

    卸载rpm包 ：rpm -e （-e 就是 eras）

    卸载deb包 ：dpkg -r（-r 就是 remove）

    3.查看已安装软件

    rpm -qa

    （-q 就是 query，a 就是 all）

    dpkg -l

    （-l 的意思就是 list）

    分页展示 more 和less

    搜索软件时，如果你不知道关键词，可以使用rpm -qa | more 和rpm -qa | less 来分页展示

    • more 是分页后只能往后翻页，翻到最后一页自动结束返回命令行，

    • less 是往前往后都能翻页，需要输入 q 返回命令行，q 就是 quit

    4.软件管家yum和apt-get

    Centos： yum
    Ubuntu ：apt-get

    5.软件下载源的配置文件

    Linux 允许我们配置从哪里下载这些软件的，地点就在配置文件里面，建议换国内的下载源

    • CentOS ：配置文件在/etc/yum.repos.d/CentOS-Base.repo里

    • Ubuntu：配置文件在/etc/apt/sources.list里

    其实无论是先下载再安装，还是通过软件管家进行安装，都是下载一些文件，然后将这些文件放在某个路径下，然后在相应的配置文件中配置一下

    例如，在 Windows 里面，最终会变成 C:\Program Files 下面的一个文件夹以及注册表里面的一些配置。

    Linux 里面会放的更散一点。例如，主执行文件会放在 /usr/bin 或者 /usr/sbin 下面，其他的库文件会放在 /var 下面，配置文件会放在 /etc 下面。

    6.压缩包安装

    下载工具 wget

    wget url ，就能从网上下载

    tar.gz格式压缩包解压 （系统自带tar）

    tar xvzf filename.tar.gz 就可以解压缩

    zip 包解压（另外下载工具）

    Centos：yum install zip.x86_64 unzip.x86_64
    Ubuntu：apt-get install zip unzip

    最后如果要运行，设置环境变量即可

    7.环境变量设置

    一次性的，设置环境变量

    export 命令仅在当前命令行的会话中管用，一旦退出重新登录进来，就不管用了

    .bashrc 或 .bash_profile 文件，登录自动执行

    在当前用户的默认工作目录，例如 /root 或者 /home/cliu8 下面，有一个.bashrc 文件(这个是隐藏文件，需要 ls -la 才能看到)

    每次登录的时候，这个文件都会运行，所以，把需要执行的命令放在这个文件里，就相当于设置了环境变量

    （也可以通过 source .bashrc 手动执行）

    通过vim编辑工具，打开.bashrc文件，写入 export 设置，保存即可完成设置环境变量

    .bash_profile是系统配置信息存储文件，写在里面的系统变量是所有用户共用的，

    而.bashrc是个人的配置信息存储文件，只是单用户有效。也就是说，配置了.bashrc后切换用户可能需要重新配置系统变量

    五、运行软件

    1.命令行里面运行

    这种模式的缺点是，一旦当前的交互命令行退出，程序就停止运行了。

    这样显然不能用来运行那些需要“永远“在线的程序。所以需要 后台模式！

    2.后台模式nohup

    这个命令的意思是 no hang up（不挂起），也就是说，当前交互命令行退出的时候，程序还要在

    • 程序不能霸占交互命令行，而是应该在后台运行。最后加一个 &，就表示后台运行。
    • 要处理的就是输出，原来什么都打印在交互命令行里，现在在后台运行了，输出到哪里呢？输出到文件是最好的。

    最终命令：nohup command >out.file 2>&1 &

    这里面，“1”表示文件描述符 1，表示标准输出，

    “2”表示文件描述符 2，意思是标准错误输出，

    “2>&1”表示标准输出和错误输出合并了。

    合并到哪里去呢？到 out.file 里。

    如何关闭后台命令：

    ps -ef |grep 关键字 |awk '{print $2}'|xargs kill -9

    • ps -ef 可以单独执行，列出所有正在运行的程序

    • grep 通过关键字找到咱们刚才启动的程序

    • awk 工具可以很灵活地对文本进行处理

      • awk ‘{print $2}’是指第二列的内容，是运行的程序 ID

    • 通过 xargs 传递给 kill -9，也就是发给这个运行的程序一个信号，让它关闭

    3.以服务的方式运行

    就像Windows的服务 ，Linux中 也有相应的服务 ，例如常用的数据库 MySQL，就可以使用这种方式运行

    Ubuntu 中 MySQL

    1. 通过 apt-get install mysql-server 的方式，安装 MySQL
    2. 然后通过命令 systemctl start mysql ，启动 MySQL
    3. 通过systemctl enable mysql，设置开机启动

    之所以成为服务并且能够开机启动，是因为在 /lib/systemd/system 目录下会创建一个 XXX.service 的配置文件，里面定义了如何启动、如何关闭。

    CentOS 的 MariaDB

    在 CentOS 里有些特殊，MySQL 被 Oracle 收购后，因为担心授权问题，改为使用 MariaDB，它是 MySQL 的一个分支

    • 通过命令yum install mariadb-server mariadb进行安装，

    • 命令systemctl start mariadb启动，

    • 命令systemctl enable mariadb设置开机启动。

    同理，会在 /usr/lib/systemd/system 目录下，创建一个 XXX.service 的配置文件，从而成为一个服务。

    六、关机和重启

    shutdown -h now：现在就关机
    reboot：重启
