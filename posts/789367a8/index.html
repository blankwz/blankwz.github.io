以下默认都是normal模式下的快捷操作！！！
有些命令只能在Vim中使用！（不能再vscode中使用）

一、基本操作

  1. Insert 和 Normal 模式互相切换：

Normal 编辑模式 —-> Insert 模式 ：i （下面有更多进入插入模式的方法）
Insert 模式 —-> Normal 编辑模式 ：ESC 键或组合键 Ctrl+[

  1. visual 可视化选择模式：
    一般是<C-v>
    windows下则是<C-q> （<C-v> 是拷贝剪贴板）
    vscode中：v
  2. 什么时候需要按回车
    以 : 开始的命令你需要输入 <enter>回车，例如 — 如果我写成 :q 也就是说你要输入 :q<enter>.
  3. 规律
  • 大写：
    Vim 编辑器经常以一对大、小写字母（如 p 和 P）来提供一对相似的功能。通常，小写命令在光标的后面进行操作，大写命令在光标的前面进行操作。

1、移动光标

hjkl (强例推荐使用其移动光标，但不必需) →你也可以使用光标键 (←↓↑→). 注: j 就像下箭头

  1. 本行移动
    • 0 → 数字零，到行头
    • $ → 到本行行尾
    • ^ → 移动光标，到本行的第一个非空字符
    • g_ → 到本行最后一个不是blank字符的位置。
    • fa （a可为任意字符）→ 到下一个字符a的位置
    • t, （逗号可为任意字符） → 到下一个字符,的前一个位置
    • 3fa （a可为任意字符）→ 在当前行查找第三个出现的字符a
    • F 和 T → 和 f 和 t 一样，只不过是相反方向。
    • dt"（"可为任意字符） → 删除所有的内容，直到遇到双引号—— "

  1. 按单词移动–>可常用

    • w → 到下一个单词的开头。
    • e → 到下一个单词的结尾。

      如果你认为单词是由默认方式，那么就用小写的e和w。默认上来说，一个单词由字母，数字和下划线组成（注：程序变量）
      如果你认为单词是由blank字符分隔符，那么你需要使用大写的E和W。（注：程序语句）

    • b → 左移光标到前一个字的开头

  2. 匹配相同单词移动—>常用
    * 当光标停留在一个单词上，* 键会在文件内搜索该单词，并跳转到下一处；
    # 当光标停留在一个单词上，# 在文件内搜索该单词，并跳转到上一处；

  1. 选中单词
    viw → 在单词任意位置，即可选择该单词
    ve → 光标在单词的开头时，可选中该单词

  2. 搜索移动
    /pattern → 搜索 pattern 的字符串（陈皓注：如果搜索出多个匹配，可按n键到下一个）

  3. 指定移动调转
    N → 到第N行
    NG → 到第 N 行 （注意命令中的G是大写的，另也可使用 : N 到第N行，如 :137 到第137行）
    gg → 到第一行。（相当于1G，或 :1）
    G → 到最后一行。

  4. 匹配括号移动
    % → 匹配括号移动，包括 (, {, [. （注：你需要把光标先移到括号上）

  5. 在屏幕中找到需要的 一页 时，可以用下面的命令快速移动光标：
    H 将光标移到屏幕上的起始行（或最上行）；
    M 将光标移到屏幕中间；
    L 将光标移到屏幕最后一行。

2、区域选择（利用Visual可视化选择模式或者其他）

区域选择<action>a<object> 或 <action>i<object>

  • action可以是任何的命令，如 d (删除), y (拷贝), v (可以视模式选择)。
  • object 可能是： w 一个单词， W 一个以空格为分隔的单词， s 一个句字， p 一个段落。也可以是一个特别的字符：”、 ‘、 )、 }、 ]。

假设你有一个字符串 (map (+) (“foo”)).而光标键在第一个 o 的位置。

  • vi” → 会选择 foo.
  • va” → 会选择 “foo”.
  • vi) → 会选择 “foo”.
  • va) → 会选择(“foo”).
  • v2i) → 会选择 map (+) (“foo”)
  • v2a) → 会选择 (map (+) (“foo”))

3、复制粘贴替换

  1. 复制：
    yy → 复制当前行到内存缓冲区，等于 ddP

    • nyy 复制 n 行内容到内存缓冲区；
    • 5yy 复制 5 行内容到内存缓冲区；
    • “+y 复制 1 行到操作系统的粘贴板；
    • “+nyy 复制 n 行到操作系统的粘贴板。

  2. 剪切/删除：

  • (剪切行)dd → 删除当前行，并把删除的行存到剪贴板里

  • (剪切单个字符) x → 向后单个字符剪切(含光标）
    nx 删除光标所指向的前 n 个字符；
    3x 删除光标所指向的前 3 个字符；
    X 向前单个字符剪切（不含光标）

  • J → 删除本行的回车符（CR），并和下一行合并。
    可在可视化选择模式中，选择多行，然后把所有的行连接起来（变成一行）

  • d → 也可以用，要搭配其他的命令使用

    • dw 删除光标右侧的字；
    • d$ 或者 D 从当前光标起删除字符直到行的结束；
    • d0 从当前光标起删除字符直到行的开始；
    • ndw 删除光标右侧的 n 个字；
    • 3dw 删除光标右侧的 3 个字；
    • db 删除光标左侧的字；
    • ndb 删除光标左侧的 n 个字；
    • 5db 删除光标左侧的 5 个字；
    • dd 删除光标所在行，并去除空隙；
    • ndd 删除（剪切） n 行内容，并去除空隙；
    • 3dd 删除（剪切） 3 行内容，并去除空隙；

  • 在命令模式下，同样可以删除正文中的内容

    • :d 删除光标所在行
    • :3d 删除 3 行
    • :.,$d 删除当前行至正文的末尾
    • :/str1/,/str2/d 删除从字符串 str1 到 str2 的所有行
    • :g/^\(.*\)$\n\1$/d 删除连续相同的行，保留最后一行
    • :g/\%(^\1$\n\)\@<=\(.*\)$/d 删除连续相同的行，保留最开始一行
    • :g/^\s*$\n\s*$/d 删除连续多个空行，只保留一行空行
    • :5,20s/^#//g 删除5到20行开头的 # 注释

  1. 粘贴
    p → 粘贴剪贴板

    • p 小写字母 p，将缓冲区的内容粘贴到光标的后面；
    • P 大写字母 P，将缓冲区的内容粘贴到光标的前面。

  2. 替换（用单个字符替换字符）

  • r → 替换字符
    • rc 用 c 替换光标所指向的当前字符；
    • nrc 用 c 替换光标所指向的前 n 个字符；
      • 5rA 用 A 替换光标所指向的前 5 个字符；

Vim常规的替换命令有 c 和 s ，结合Vim的其他特性可以实现基础的替换功能，不过替换命令执行以后，通常会由 编辑模式 进入 插入模式 ：(vscode无法使用)

  • s 用输入的正文替换光标所指向的字符；

  • S 删除当前行，并进入编辑模式；

  • ns 用输入的正文替换光标右侧 n 个字符；

  • nS 删除当前行在内的 n 行，并进入编辑模式；

  • cw 用输入的正文替换光标右侧的字；

  • cW 用输入的正文替换从光标到行尾的所有字符（同 c$ )；

  • ncw 用输入的正文替换光标右侧的 n 个字；

  • cb 用输入的正文替换光标左侧的字；

  • ncb 用输入的正文替换光标左侧的 n 个字；

  • cd 用输入的正文替换光标的所在行；

  • ncd 用输入的正文替换光标下面的 n 行；

  • c$ 用输入的正文替换从光标开始到本行末尾的所有字符；

  • c0 用输入的正文替换从本行开头到光标的所有字符。

  • 利用 :s 命令可以实现字符串的替换

    • :%s/str1/str2/ 用字符串 str2 替换行中首次出现的字符串 str1
    • :s/str1/str2/g 用字符串 str2 替换行中所有出现的字符串 str1
    • :.,$ s/str1/str2/g 用字符串 str2 替换正文当前行到末尾所有出现的字符串 str1
    • :1,$ s/str1/str2/g 用字符串 str2 替换正文中所有出现的字符串 str1
    • :g/str1/s//str2/g 功能同上
    • :m,ns/str1/str2/g 将从m行到n行的str1替换成str2

从上述替换命令可以看到：
1. g 放在命令末尾，表示对搜索字符串的每次出现进行替换,不止匹配每行中的第一次出现；不加 g，表示只对搜索字符串的首次出现进行替换；g 放在命令开头，表示对正文中所有包含搜索字符串的行进行替换操作;
2. s 表示后面跟着一串替换的命令；
3. % 表示替换范围是所有行，即全文。

在Vim中统计当前文件中字符串 str1 出现的次数，可用替换命令的变形：:%s/str1/&/gn

4、撤销重做

u 撤销
ctrl+r 重做

5、重复

  • . → (小数点) 可以重复上一次的命令

  • 数字 + <command> → 重复某个命令N次

例子：

  • 2dd → 删除2行
  • 3p → 粘贴文本3次
  • 100idesu [ESC] → 输入100，等于100次；输入i，插入；desu，是字符。esc按键，退出编辑模式。则立马插入100个desu
  • . → 重复上一个命令
  • 3. → 重复 3 次上次的命令

或者宏录制

6、进入插入模式

  • i 在光标左侧插入正文
  • a 在光标右侧插入正文
  • o 在光标所在行的下一行增添新行
  • O 在光标所在行的上一行增添新行
  • I 在光标所在行的开头插入
  • A 在光标所在行的末尾插入

7、打开/保存/退出/改变文件(Buffer)

:wq → 存盘 + 退出 (:w 存盘, :q 退出) （陈皓注：:w 后可以跟文件名）

  • :e path_to_file/filename → 打开一个文件

  • :w → 保存当前编辑的文件（单词 write 的缩写）

    • :w file_temp 将当前文件另存为 file_temp

  • :saveas <path/to/file> → 另存为 <path/to/file>

  • :x， ZZ 或 :wq → 保存并退出 (:x 表示仅在需要时保存，ZZ不需要输入冒号并回车)

  • :q! → 退出不保存 :qa! 强行退出所有的正在编辑的文件，就算别的文件有更改。

  • :q → 在未作修改的情况下退出；

  • :wq → 保存并退出则可以讲两条命令结合起来使用（注意命令顺序，先保存，后退出）

  • :bn 和 :bp → 你可以同时打开很多文件，使用这两个命令来切换下一个或上一个文件。（陈皓注：我喜欢使用:n到下一个文件）

8、缩进

  • 进入可视化选择（ <C-v>，windows是<C-q>，vscode是V）
  • 方向键选择行
  • < 或 > → 左右缩进
  • = → 自动给缩进

二、高级

1、移动光标命令+命令+移动光标命令

<start position><command><end position>

例如 0y$ 命令意味着：

  • 0 → 先到行头
  • y → 从这里开始拷贝
  • $→ 拷贝到本行最后一个字符

2、块操作

  1. 块操作快捷键一般是<C-v>

    • windows下则是 <C-q> （ <C-v> 是拷贝剪贴板）
    • vscode中，直接通过按 V键进入可视化选择

  2. 在所有被选择的行的，前面，加内容

    • 0 → 到行头
    • <C-q> → 开始块操作
    • <C-d> → 向下移动 (你也可以使用hjkl来移动光标，或是使用%，或是别的)
    • (大写的)I-- [ESC] → I是插入，插入“–”，按ESC键来为每一行生效。

  3. 在所有被选择的行的，后面，加内容

    • <C-v>
    • 选中相关的行 (可使用 j 或 <C-d> 或是 /pattern 或是 % 等……)
    • $ 到行最后
    • A, 输入字符串，按 ESC。

3、自动提示

在 Insert 模式下，你可以输入一个词的开头，然后按 <C-p>或是<C-n>，自动补齐功能就出现了

4、宏录制 （操作序列，比如1-100）

  • qa 把你的操作记录在寄存器 a。
  • 于是 @a 会replay被录制的宏。
  • @@ 是一个快捷键用来replay最新录制的宏。

示例
在一个只有一行且这一行只有“1”的文本中，键入如下命令：

  • qaYp<C-a>q→
    • qa 开始录制
    • Yp 复制行.
    • <C-a> 增加1. —->1变成了2（用于增加序列）
    • q 停止录制.
  • @a → 在1下面写下 2
  • @@ → 在2 正面写下3
  • 现在做 100@@ 会创建新的100行，并把数据增加到 103.

5、行号与文件

  • 命令模式（ 通过 ： 进入命令模式）

  • 命令模式下，可以规定命令操作的行号范围。

    • 数值用来指定绝对行号；
    • 字符“.”表示光标所在行的行号；
    • 字符“，”表示 至 ，区间
    • 字符符“$”表示正文最后一行的行号；

  • 简单的表达式，例如“.+5”表示当前行往下的第 5 行。例如：

    • :345 将光标移到第 345 行
    • :345w file 将第 345 行写入 file 文件
    • :3,5w file 将第 3 行至第 5 行写入 file 文件
    • :1,.w file 将第 1 行至当前行写入 file 文件
    • :.,$w file 将当前行至最后一行写入 file 文件
    • :.,.+5w file 从当前行开始将 6 行内容写入 file 文件
    • :1,$w file 将所有内容写入 file 文件，相当于 :w file 命令

  • 在命令模式下，允许从文件中读取正文，或将正文写入文件。例如：

    • :w 将编辑的内容写入原始文件，用来保存编辑的中间结果
    • :wq 将编辑的内容写入原始文件并退出编辑程序（相当于 ZZ 命令）
    • :w file 将编辑的内容写入 file 文件，保持原有文件的内容不变
    • :a,bw file 将第 a 行至第 b 行的内容写入 file 文件
    • :r file 读取 file 文件的内容，插入当前光标所在行的后面
    • :e file 编辑新文件 file 代替原有内容
    • :f file 将当前文件重命名为 file
    • :f 打印当前文件名称和状态，如文件的行数、光标所在的行号等

6.Vim正则表达式，vscode中好像不能用

参考：

  • 《正则表达式30分钟入门》
  • 进阶的Vim正则表达式还有对Magic 模式的介绍，可以参考《Vim正则表达式详解》

当给Vim指定搜索字符串时，可以包含具有特殊含义的字符。包含这些特殊字符的搜索字符串称为正则表达式（Regular Expressions）

下表给出大多数特殊字符和它们的含义：
-^ 放在字符串前面，匹配行首的字；

  • $ 放在字符串后面，匹配行尾的字；
  • \< 匹配一个字的字头；
  • \> 匹配一个字的字尾；
  • . 匹配任何单个正文字符；
  • [str] 匹配 str 中的任何单个字符；
  • [^str] 匹配任何不在 str 中的单个字符；
  • [a-b] 匹配 a 到 b 之间的任一字符；
  • * 匹配前一个字符的 0 次或多次出现；
  • \ 转义后面的字符。
