    Windows下通过Vagrant安装Linux系统进行学习

    一、工具

    1. VirtualBox:

      • VirtualBox 是一款开源虚拟机软件

      • 下载

    2. Vagrant:

      • Vagrant是一个基于Ruby的工具，用于创建和部署虚拟化开发环境。
    • 拿VirtualBox举例，VirtualBox会开放一个创建虚拟机的接口，Vagrant会利用这个接口创建虚拟机，并且通过Vagrant来管理，配置和自动安装虚拟机
      • 下载 选择系统版本 （需搭梯子，下载速度才会起来）

    下载并安装他们

    Vagrant安装完后，需重启

    选择一个系统镜像，装进VB，这里我们主要是Vagrant教程

    二、下载镜像

    1. 在 Vagrant下筛选VB的包，找到你想要的系统镜像

    推荐：到 清华开源站镜像下载,比如Ubuntu的镜像,（找后缀为box的包）

    1. 创建个存放目录，比如 D:/workspace

    2. 使用 Vagrantfile 或者 包的名称来进行添加镜像 （只是添加）

    3. 如果想下载开源镜像站的话

    4. 安装镜像： vagrant up

    如果觉得下载速度慢（需搭梯子），可以网上找镜像box文件，自行下载到本地目录

    • 如果你有梯子，无法开启全局模式的情况下，可以在 vagrant up 时，看到box 的文件地址，再通过浏览器下载

    总结：

    三、Vagrant配置

    Vagrant 的核心配置文件是 Vagrantfile

    • 官方教程：文档

      • 端口映射教程
      • 私有网络教程
      • 共有网络教程

    • 以下所有代码操作都是编辑，创建的文件夹下的Vagrantfile

    设置本机与虚拟机同步的目录

    • 例：config.vm.synced_folder "d:/workspace", "/workspace"

    设置网络端口转发, 也是就虚拟机中的80端口映射为宿主机上的8080端口

    • config.vm.network :forwarded_port, guest: 80, host: 8080
    • 方法2：VirtualBox中设置端口转发，缺点是，每次通过vagrant reload命令重启虚拟机以后失效
      • 打开VB —- 设置 —网络 —高级 —-端口转发 —添加转发规则 —保存
        • 协议：TCP
        • 主机IP：127.0.0.1
        • 主机端口：8080
        • 子系统端口：80
    • 优点：简单、容易理解、容易实现外网访问虚拟机。
    • 缺点：需映射很多端口时较麻烦、不支持在宿主机器上使用小于1024的端口来转发(如:不能使用SSL的443端口来进行https连接)

    设置私有网络地址

    • 固定IP：config.vm.network :private_network, ip: “192.168.10.10”

    • 或者固定IP：config.vm.network "private_network", type: "dhcp"

    • 这种网络配置下，只有主机可以访问虚拟机，如果多个虚拟机设置定在同一个网段也可以相互访问，当然虚拟机也是可以访问外部网络的

    • 优点：安全，只能自己访问

    • 缺点：因私有原有，所以其他团队成员不能和你协作

      设置公有网络地址

    • 固定IP：config.vm.network :public_network, ip: “192.168.10.10”

    • 桥接网卡：config.vm.network "public_network", bridge: "en1: Wi-Fi (AirPort)"

    • 这种配置下，虚拟机享受实体机一样的待遇，一样的网络配置，vagrant 1.3版本以后这种配置也支持设定

    • 优点：方便团队协作，别人可以访问你的虚拟机。

    • 缺点：需要有网络，有路由器分配IP

    错误问题

    Q1：Message: undefined local variable or method "\u201Cgxkon\u201D"'

    A1：检查Vagrantfile文件，格式错误，是否用了中文的符号。比如 “”，一般是账号密码，从其他地方复制过来的~

    Q2：vagrant 卡在 SSH auth method: password 或者 SSH auth method: private key

    A2：参考：安装问题汇总 这个虽然没用（不适合我），但是也可以是个思路.

    四、 Vagrant常见命令

    命令 解释
    vagrant box list 查看目前已有的box
    vagrant box add 新增加一个box
    vagrant box add {box_name} {file_path} 添加box到本地vagrant环境
    vagrant box remove box-name 删除指定box
    vagrant box repackage box-name 重新打包本地环境中指定的box
    vagrant init [box-name] 初始化配置vagrantfile ，生成Vagrantfile文件
    vagrant up 在初始化完的文件夹内，启动虚拟机
    vagrant ssh ssh登录虚拟机，退出方法：exit 或者 Ctrl+D
    vagrant suspend 挂起虚拟机
    vagrant reload 重启虚拟机 （修改配置文件后，重启虚拟化开发环境）
    vagrant halt 关闭虚拟机
    vagrant status 查看虚拟机状态
    vagrant destroy 删除虚拟机
    vagrant ssh-config 检查ssh配置
    vagrant help 查看更多命令
