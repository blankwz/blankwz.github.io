Quantumult X教程

作者Github：地址

以下简称

Quantunmult X右下角的图标 — > QX图标

一、介绍

Quantumult X 是 Quantumult 作者的另一款网络代理工具。

  • 界面设计更漂亮，支持目前流行的 JavaScript 脚本

1. What

他除了代理，还可以具体做什么

  • 解决本地 DNS 可能带来的干扰
  • 解决部分网站跳转问题
  • 可突破部分内网限制（公司、学校）
  • 拦截部分挖矿 JS 插件
  • 拦截常用应用程序及网页的行为分析
  • 拦截常用应用程序及网页的数据统计
  • 拦截常用应用程序及网页的隐私跟踪
  • 拦截各大购物网站的运营商劫持
  • 拦截 Content Security Policy 劫持
  • 拦截 CNNIC 根证书劫持
  • 屏蔽部分应用程序的启动广告
  • 屏蔽部分运营商劫持网页弹出的流量统计
  • 屏蔽部分运营商劫持网页弹出的漂浮球广告
  • 屏蔽常用视频广告
  • 屏蔽常用网站广告、其他流媒体网站广告
  • 屏蔽法轮功等反华势力网站
  • 所有国内网站直线连接
  • Apple 服务加速（App Store、Apple Music、Apple流媒体、iCloud备份、iCloud Drive、iTunes 等）
  • 国外常用网站加速（Google/Youtube/Twitter/Facebook/instagram/wikipedia/Github 等）

2. 对于此类软件的进阶过程

  1. 青铜：使用订阅节点，可以翻
  2. 白银：会使用网络大佬们的脚本、规则
  3. 黄金：除了运用自如脚本和规则，更了解脚本和规则的原理，可以整合搬运各种脚本和规则
  4. 白金：可以简单编写脚本和规则
  5. 大佬：脚本和规则，手到擒来。

能达到手到擒来级别，需要的知识，包含但并不限于网页前端、网页后端、正则表达式、破解

3.代理模式

Quantumult X 有三种代理模式：

  • 全部代理
  • 全部直连
  • 规则分流 （推荐）

如何开启规则分流模式： 请点击软件右下角的功能按钮，然后在新窗口点击右上角三个点的标志，选择”规则分流“。

4. QX大致分为四种大功能

  • 节点
  • 分流(规则)
  • 重写(规则)
  • 强大的脚本

二、四大功能

1. 节点

用代理的都懂，不用讲了吧

它支持 ss，ssr 订阅，以及 Quanx 格式的 vmess / https / trojan 订阅

  • 引用（订阅）的节点，对应配置文件的 [server_remote] 部分

2. 分流（策略）

规则分流模式下，网络请求的走向由分流规则中的所匹配到的策略决定

我们建立了不同的规则（道路)，它会检测你访问的网址的域名或者DNS

主要功能

  • 对于梯子来说，自定义你的某些网络请求是走哪些线路、节点、流量
    • 指定奈飞线路
    • 指定TikTok线路
    • 如果定义为电报（Telegram）的数据访问，就会把数据归到电报策略节点位
    • 一些屏蔽国内IP的应用，切换线路
  • 对于破解来说，自定义某些域名的直连、拒绝访问等等功能
    • 去广告 ：当它检测到广告数据（域名、资源等）时，就进行拦截

策略组

策略组的意义在于精细化控制

  • 策略组里选择 PROXY 意味着和默认节点保持一致
  • 选择 DIRECT 意味着将使用国内网络直接进行连接

关于策略组的深层理解，请参考F大的wiki

添加策略组方法（配置文件方法）

在配置文件中 的 [policy] 中添加

机场订阅地址转换策略组方法

在 资源列表（节点）中，向右划动你想转换的节点列表 - 点击更多 - 选择一个策略即可 - 到主页中查看，是否多了一个策略组

  1. QX图标 - 找到节点模块下引用，点击
  2. 往右滑任意订阅，点击更多即可创建策略
  3. 三个策略选择一个

静态策略-手动选择： 跟PROXY一样；（需至少配置一个节点）
*健康检查-自动选择：** 从第一个节点进行可用性检查，如果可用即选择第一个节点，不可用则继续检查直到节点可用；（*需至少配置两个节点）

负载均衡-轮询调度： 轮流调用节点使用，如果使用该策略则访问谷歌时你的ip是一直变化的；（*需至少配置两个节点）

每个策略下都可以有很多节点，这些节点是由你自己选择的（可长按策略图标进行增减），PROXY默认拥有所有订阅而来的节点，随每次更新节点订阅时而有所增减，其中的节点需要手动选择使用；

分流规则搭配策略组

  1. QX图标 - 分流下 - 引用
  2. 左滑任意分流订阅链接 - 编辑 - 开启策略偏好 - 选取刚刚新建的策略组

为单条分流规则绑定策略组

  1. QX图标 - 分流 - 添加
  2. 填写相应类型，参数，策略选择刚刚创建的即可

策略更改节点方法

策略组模块上有白色环形标志的代表可以长按以添加或删除其中的节点，长按策略图标

  • 把需要的节点添加上去
  • 不需要的删除掉

3. Rewrite 重写 与 MitM

这两个功能主要作用是辅助广告拦截。可以根据自己的情况按需开启

用于修改HTTP或HTTPS请求与响应

4. 脚本

这就更厉害了，自动执行某些脚本，达到自动化目的

  1. 获取Cookie
  2. 访问API
  3. 签到、领取金币等等

  • 一个名为xxx.js 的文件(它可以是本地文件，也可以是别人放在GitHub的远程)

  • 有了JS文件之后，你当然还需要告诉 Quantumult X，你要如何使用这个xxx.js

    • 写法的通用表达 ： 匹配网址 url 复写类型 脚本文件

  • 这个语句是一句 rewrite，可以放在本地配置(rewrite_local)，也可以放远程引用(rewrite_remote)

  • 各类型的具体说明，参见作者: https://github.com/crossutility/Quantumult-X

由于某些人提供制作破解 VIP 的脚本，甚至在作者稍加限制之后(设备ID)，仍旧自作聪明的公然想办法破解，甚至拿来当作服务出售。因此开发者一度取消了远程 js 脚本。

但在版本 1.0.12后，作者重新开放了 远程脚本的一键更新 ，

但需满足一个条件：购买时间>30天 否则，你依旧可以使用远程 js 脚本，但只能一个个手动更新

三、使用

软件基本操作

Quantunmult X 中有很多隐藏操作，一般分为

  • 点按
  • 长按 ，很多时候找不到设置方法，可以长按试试

1.简单 - 使用节点

点击 - QX图标 - 节点

  • 添加：添加单个SS节点
  • 引用（订阅）： 添加订阅信息（机场节点）
  • SS URI
  • 扫码

节点测速方法:

单个测速：长按节点名称 - 网页响应测试

全部测速：进入某个策略组 - 下拉 - 即可测速

2. 中级 - 使用（远程）懒人包配置文件

预配置文件的意义在于帮助新手用户（或懒人用户）一键导入软件运行所必需的一些配置，包括分流规则、策略、策略组、DNS、MitM、Rewrite规则等。导入预配置文件后，仅需要手动添加节点即可享受这款软件带来的畅快科学上网体验。

1）添加大佬们的配置文件

  1. 找个大佬的懒人配置文件

  2. QX图标 - 配置文件 - 下载 - 输入远程配置文件raw地址 （会覆盖当前所有配置，包括节点）

  3. 保存

2）安装证书

  1. 点击QX图标 - 往下拖- 打开 重写 和 MitM
    1. 点击MitM的 生成证书，再点击安装证书 （描述文件）
      • 若提示已经有了有效的配置文件，则点击 配置证书
      • 设置 - 顶部 - 已下载的证书 - 点击后，安装
    2. 在设置 - 通用 - 关于本机 - 证书信任设置 中可以找到，刚刚安装的Quantunmult信任证书 - 信任它
  2. QX图标 - 最下面的其他设置 - VPN - 打开 始终开启 （为了方便测试）
  3. 点击 QX图标，添加你的节点
  4. 添加完成后，打开右上角VPN开关

重新生成MitM的证书方法（此方法适用于证书无效时急救）

  1. 到 配置文件 中，最下方的 [mitm] - 删除其下方内容
  2. 到 QX图标 - Mitm - 生成证书
  3. 再配置证书，下载证书，后面就是安装证书和信任证书了

四、文件

1. 文件存放

Quantumult X的文件都在 IOS的文件app中，下载完Quantumult X，打开IOS的文件

  1. 可以看到iCloud Drive和 我的iPhone中都自动创建了Quantunmult X文件夹，其中含有

    • Images：图标图片文件
    • Profiles：配置关联地址
    • Scripts：脚本文件
    • 顾名思义，若想让 Quantumult X 使用哪些资源，就要在改资源的文件夹下有文件。

  2. 至于到底存放在iCloud Drive还是 我的iPhone，看你自己选择

    • 若用iCloud Drive，需要在设置 - 其他设置 - 开启资源文件夹的iCloud

2. 配置文件 .conf

Quantumult X 有很多配置，各个配置各有分工

  • Quantumult X 配置文件
  • 重写 脚本配置文件

配置文件分为远程配置文件及本地配置文件，任选其一！！！！

  • 远程配置文件：通过 QX图标 - 配置文件 - 下载 - 输入远程配置文件raw地址
  • 本地配置文件：远程挂载方式下载到 苹果的 文件app中

Quantumult X 预配置文件

  • 是一个完整的配置文件，包含 Quantumult X 的所有配置项目，如 分流，策略及图标设置，脚本等等
  • 打开 - [Quantumult X] - [配置文件] - [编辑] 即可查阅当前你已配置的各个项目
  • 一次性只能用一个配置文件
    • 用了远程配置文件就不能使用本地配置文件
    • 新增一个配置文件需要重新配置证书和设置

重写 脚本配置文件

  • 重写 为JS脚本配置文件，该文件内含：
    • hostname配置
    • 各种JS脚本详细配置
  • JS脚本配置文件对应位置在：[Quantumult X] - [配置文件] - [编辑] - [rewrite_remote] - 粘贴脚本配置文件地址

3. 远程QX配置文件

4. Working Copy远程挂载到本地配置文件

Working Copy 是 iOS 平台上的一款全功能 Git 客户端，其支持新建本地仓库，并与 GitHub，GitLab，BitBucket，甚至是自建 Git 服务器上的远程仓库同步。

Q：为什么使用它？

A：我们需要把大佬们的 各种策略下载到iPhone的QX本地文件夹中，通过Working Copy可以方便的把Github的内容下载到本地，并且方便的更新。

  1. IOS下载Working Copy
  2. 到文件app中的 iCloud Drive或者 我的iPhone - Quantunmult X - Script文件夹内- 长按空白处建立个空文件夹
  3. 打开Working Copy ， 右上角 + ，选择 Setup Synced Directory - 找到刚刚建立的空文件夹
  4. 进去后 - 点击 Repository - Add Remote - 把脚本库的地址复制过来
    • 脚本库地址一定是带git后缀的
      • Github - Code下载按钮 - HTTPS - 复制地址
  5. 关掉 Allow Push，点击保存
  6. 点击刚刚进里的 Remote，点击左上角的Fetch，等待
    1. 如果是Nzw9314大佬的脚本，第一次一般都会报错 ：Reference 'refs/heads/maste'r not found，没关系，后退
    2. 点击 Branch - 选择默认分支
    3. 点击Checkout
  7. 完成后，后退，可以看到都出来了
  8. 去刚刚 文件app中建立的空文件夹内，是否已经全部下载完成 （完成远程挂载）
    • Nzw9314大佬的脚本的话，点击文件夹内的名为Quantunmult X配置文件 - （自动打开了Quantunmult X）右上角保存Working Copy

五、脚本

1. 要素

一个完整的脚本使用，需要

  • 正确的复写语句(rewrite)
  • 正确的域名解析（hostname,中间人攻击）
  • 正确的本地脚本放置路径(xx.js)

一般好的脚本js文件，里面都会有完整的使用说明，按照说明操作

2. js脚本文件位置

请大佬们 千万别 直接把js脚本链接直接丢进rewrite引用，然后还跑来问“怎么报错？”

同时，放对位置后，需要去 rewrite_local里添加引用

  1. 可以放 github 远程使用（raw 链接），（仅限新TF版本可以使用远程）
  2. 也可以放入本机文件夹中 /On My iPhone/Quantumult X/Scripts 目录下
  3. 当然也可以放入iCloud Drive下的Quantumult X/Scripts 目录

3.脚本使用

  1. 如何引用 JS脚本配置文件：[Quantumult X] - [重写] - [引用] - [+]

    • 有的脚本需要把hostname 添加到 配置文件中（ QX图标 - 配置文件 - 编辑）

  2. 脚本执行： QX图标 - 构造请求 - 找到对应脚本 - 向右滑动 - 执行按钮

  3. 脚本需要更新才能使用

    • 如果你购买Quantumult X还不足30天，脚本是无法一键全部更新的

  4. 各脚本具体使用方法，直接点击脚本，查看脚本

    • 一般脚本都会有详细的使用说明

  5. 部分脚本的运行，需要手动加入Cookies

  6. 脚本可以定期执行，在请求列表右上角 的时钟按钮，开关定期执行功能

    • 具体的执行时间，在代码可以看到

  7. 不需要的脚本，建议禁用（注释掉）

    • 注释方法： 在QX配置文件中，行前面 添加 半角;即可注释

当需要抓包（获取Cookie）时，需要开启 调试 （QX图标 - 调试）的开关

4. 问题

某些脚本js文件没有效果

  1. QX图标 - 重新- 搜索
  2. 输入 js脚本关键字，比如jd
  3. 更新它

淘宝比价没有用

  1. QX图标 - 重新- 搜索
  2. 输入 taobao
  3. 找到比价的js，更新
  4. 试试看
  5. 如果还是不行，就卸载了重新安装

九、技巧

1. 高效管理节点

为什么需要

机场的订阅地址都是，一个订阅地址含有各种地区（日本、美国等）或功能（奈飞等）的节点列表

这样做设置时很麻烦，需要一个个的给他们分类

那能不能把订阅地址的节点列表，分开来，让他能实现，一个订阅地址一个地区（功能），这样的话，设置地区或功能时，只需要统一设置成一个订阅地址就可以了。

  • 机场订阅地址转换策略组方法
