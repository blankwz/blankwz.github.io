    Markdown语法概览

    注意：Markdown语法在每个Markdown编辑器都会有细微的语法和功能上的差别
    不是所有语法都适用

    一、我不太常用，但很有用

    1.待办事项

    使用 - [ ] 和 - [x] 语法可以创建复选框，实现 todo-list 等功能。例如：

    -------------------效果----------------------

    • 已完成事项
    • 待办事项1
    • 待办事项2

    2.删除线

    -------------------效果----------------------
    不要删我

    3.toc目录

    直接写 [toc]即可

    [TOC] # 就不演示了，太长了

    4.跳转！！！（Typora内部跳转）

    Typora语法方法 - Typora的跳转都必须按住CTRL，跳转

    Hexo跳转（Post_link）语法

    • 一个是herf标签，一个是正常的[]()
    • 都只能跳转到标题#
    • 空格可以用-代替（比如：## 跳 到 这 里 ，可以用 [我想跳转](#跳-到-这-里)）
      • 我想跳转

    4.1 本文节点-锚点跳转

    本文节点-锚点跳转

    这里为什么不用 4.1 ，因为vscode不支持 4.1-本文节点-锚点跳转 这样的跳转，而Typora支持

    -------------------效果----------------------
    我想跳转到底部
    Link to Google

    4.2 跳转本地文件及锚点

    -------------------效果----------------------
    打开本目录其他文件 (链接到当前目录下文件，直接输入文件名和后缀)
    打开本目录其他文件并跳转到指定锚点 (Typora不支持文件锚点跳转，VSCODE可以)

    5. 按键

    -------------------效果----------------------

    Ctrl + F

    二、常用

    1.字体样式编辑

    - Markdown 使用星号（*）和底线（_）作为标记强调字词的符号
- 被 * 或 _ 包围的字词会被转成用 <em> 标签包围
- 用两个 * 或 _ 包起来的话，则会被转成 <strong>

    -------------------字体样式效果----------------------
    演示粗体
    演示斜体

    这是 H1

    这是 H2

    。。。

    这是 H6

    single asterisks
    single underscores
    double asterisks
    double underscores

    • 也可以直接插在文字中间
      但是如果你的 * 和 _ 两边都有空白的话，它们就只会被当成普通的符号
      unfriggingbelievable

    2.列表

    2.1 无序列表使用星号、加号或是减号作为列表标记

    -------------------列表效果----------------------

    • Red
    • Green
    • Blue

    2.2 段落中使用（让列表看起来更漂亮，你可以把内容用固定的缩进整理好）

    包含多个段落，每个项目下的段落都必须缩进 4 个空格或是 1 个制表符

    -------------------效果----------------------

    • Lorem ipsum dolor sit amet, consectetuer adipiscing elit.
      Aliquam hendrerit mi posuere lectus. Vestibulum enim wisi,
      viverra nec, fringilla in, laoreet vitae, risus.
    • Donec sit amet nisl. Aliquam semper ipsum sit amet velit.
      Suspendisse id sem consectetuer libero luctus adipiscing.

    2.3 有序列表则使用数字接着一个英文句点

    -------------------效果----------------------

    1. Bird
    2. McHale
    3. Parish

    2.4 有序列表用ol 和li

    -------------------效果----------------------

    1. McHale
    2. Bird
    3. Parish
    ---

    2.5 如果要在列表项目内放进引用，那 > 就需要缩进

    -------------------效果----------------------

    • A list item with a blockquote:

      This is a blockquote
      inside a list item.

    2.6 如果要放代码区块的话，该区块就需要缩进两次，也就是 8 个空格或是 2 个制表符

    -------------------效果----------------------

    • 一列表项包含一个列表区块：

      <代码写在这>

    2.7 行首出现数字-句点-空白,可以在句点前面加上反斜杠

    -------------------效果----------------------
    1986. What a great season.

    3.代码区块

    3.1 立代码区块很简单，只要简单地缩进 4 个空格或是 1 个制表符就可以

    -------------------效果----------------------
    这是一个普通段落：

    这是一个代码区块。

    Here is an example of AppleScript:

    tell application "Foo"
    beep
end tell

    3.2 指定代码（有颜色）

    -------------------效果----------------------

    3.3 如果要标记一小段行内代码，你可以用反引号把它包起来（`）

    -------------------效果----------------------

    Use the printf() function.

    3.4 如果要在代码区段内插入反引号，你可以用多个反引号来开启和结束代码区段

    -------------------效果----------------------
    There is a literal backtick (`) here.

    A single backtick in a code span: `

    A backtick-delimited string in a code span: `foo`

    3.5 在代码区段内，& 和方括号都会被自动地转成 HTML 实体，这使得插入 HTML 原始码变得很容易

    -------------------效果----------------------

    Please don’t use any <blink> tags.
    &#8212; is the decimal-encoded equivalent of &mdash;.

    4.分割线

    • 一行中用三个以上的星号、减号、底线来建立一个分隔线
    • 行内不能有其他东西。你也可以在星号或是减号中间插入空格 -------------------效果----------------------
      就是多条分割线

    5.链接

    • 两种形式的链接语法： 行内式和参考式两种形式
    • 链接文字都是用 [方括号] 来标记
    • 链接辨别标签可以有字母、数字、空白和标点符号，但是并不区分大小写

      5.1 行内式

    • 只要在方块括号后面紧接着圆括号并插入网址链接即可
    • 如果你还想要加上链接的 title 文字，只要在网址后面，用双引号把 title 文字包起来即可

    This is an example inline link.

    This link has no title attribute.

    如果你是要链接到同样主机的资源，你可以使用相对路径

    -------------------效果----------------------

    See my About page for details.

    5.2 参考式

    • 在链接文字的括号后面再接上另一个方括号
    • 在第二个方括号里面要填入用以辨识链接的标记
    • 选择性地在两个方括号中间加上一个空格
    • 如果显示失败，可能需要在上一行和下一行都留一空白行 -------------------效果----------------------

    This is [an example] id reference-style link.
    - 在文件的任意处，你可以把这个标记的链接内容定义出来，上面的id

    5.2.1 链接内容定义的形式为：
    • 方括号（前面可以选择性地加上至多三个空格来缩进），里面输入链接文字
    • 接着一个冒号
    • 接着一个以上的空格或制表符
    • 接着链接的网址
    • 选择性地接着 title 内容，可以用单引号、双引号或是括弧包着 -------------------效果----------------------
    5.2.2 隐式链接标记
    • 让你可以省略指定链接标记
    • 链接标记会视为等同于链接文字
    • 要用隐式链接标记只要在链接文字后面加上一个空的方括号

    -------------------效果----------------------

    Google1

    Visit Daring Fireball for more information.

    5.2.3 链接的定义可以放在文件中的任何一个地方
    • 偏好直接放在链接出现段落的后面，你也可以把它放在文件最后面，就像是注解一样。

    -------------------效果----------------------
    I get 10 times more traffic from Google 1 than from
    Yahoo 2 or MSN 3.

    改成用链接名称的方式写

    -------------------效果----------------------
    I get 10 times more traffic from Google than from
    Yahoo or MSN.

    5.2.4 脚注

    -------------------效果----------------------

    这是一个链接到谷歌的^脚注。

    5.3 自动链接

    • 只要是用方括号包起来， Markdown 就会自动把它转成链接。一般网址的链接文字就和链接地址一样

    -------------------效果----------------------
    http://example.com/
    address@example.com

    6.图片

    • 同样也允许两种样式： 行内式和参考式。
    • 方法：
      • 一个惊叹号 !
      • 接着一个方括号，里面放上图片的替代文字
      • 接着一个普通括号，里面放上图片的网址，最后还可以用引号包住并加上 选择性的 ‘title’ 文字。

    行内式

    -------------------效果----------------------

    参考式

    -------------------效果----------------------
    id: url/to/image “Optional title attribute”

    没有办法指定图片的宽高，如果你需要的话，你可以使用普通的 <img> 标签。

    7.反斜杠

    利用反斜杠来插入一些在语法中有其它意义的符号

    Markdown 支持以下这些符号前面加上反斜杠来帮助插入普通的符号：

    \   反斜线
`   反引号
*   星号
_   底线
{}  花括号
[]  方括号
()  括弧
#   井字号
+   加号
-   减号
.   英文句点
!   惊叹号

    8.表格

    1. |、-、:之间的多余空格会被忽略，不影响布局。
    2. 默认标题栏居中对齐，内容居左对齐。
    3. -:表示内容和标题栏居右对齐，:-表示内容和标题栏居左对齐，:-:表示内容和标题栏居中对齐。
    4. 内容和|之间的多余空格会被忽略，每行第一个|和最后一个|可以省略，-的数量至少有一个

    -------------------效果----------------------
    | 左对齐标题 | 右对齐标题 | 居中对齐标题 |
    | :——| ——: | :——: |
    | 短文本 | 中等文本 | 稍微长一点的文本 |
    | 稍微长一点的文本 | 短文本 | 中等文本 |
