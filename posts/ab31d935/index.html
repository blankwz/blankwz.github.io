斐讯N1刷各种系统

N1优缺点和玩法

N1的优势在于：

  • ARM64主流架构，真64位linux系统，docker随便玩。（Armbian输出：Linux aml 5.0.2-aml-s905 #5.77 SMP PREEMPT Mon Apr 1 17:41:33 MSK 2019 aarch64 GNU/Linux）
  • 真千兆有线网口，2.4/5G双频wifi，可以作热点。
  • 自带8G的emmc存储，linux系统直接刷到盒子里，不用额外插SD卡，I/O性能好。
  • CPU 4核，2G内存。

唯一缺点：

  • USB2.0，显然是不配套千兆网口带宽的，磁盘将成为了下载电影的瓶颈。
  • 需要精力折腾，因为是小众社区来支持N1硬件适配到Armbian系统，安装过程缺乏标准化的方案。

N1的几种玩法

  • 电视盒子
  • 游戏盒子
  • BT/PT下载机
  • Linux小主机（有了Linux，玩法就更多了）
  • 路由器

事前准备

  • 斐讯N1一台

  • 配套固件镜像文件

  • 双公头 USB 刷机线一根

  • 8GB 以上U盘一个

  • HDMI线一根

  • USB 键盘鼠标一套

  • 刷机工具

    • USB刷录工具:USB Image Tool
    • ADB调试工具
    • 固件:Openwrt、Armbian
    • 降级工具：webpad

连接准备

  1. 先断网（避免N1连接网络时自动更新）
  2. 路由器连接N1，通网
  3. N1的HDMI线连接电视和鼠标
  4. USB刷机线一头连接N1，一头连接电脑USB口
  5. 通电开机，会进入斐讯自带的安卓系统

U盘先不连接，如果需要连接，请连接离HDMI口最近的那个USB口

一、降级

一般新N1版本比较新，需要先把版本降级到2.19以下

一台N1这辈子只需要做一次降级操作，后续不需要再研究这个问题了

  1. N1盒子开启ADB调试

    • 开机后，电视上查看固件版本
    • 点击 固件版本的版本号，4次
    • 下方出现 ADB调试开启 或者打开adb字样，点击
    • 记下IP

  2. 电脑端安装ADB调试工具

    • MAC在终端：brew cask install android-platform-tools

  3. 手动降级或者降级工具降级

    • 降级工具降级，最简单，输入IP，点降级即可
    • 手动降级: （参考webpad大神）
      • 电脑（adb）连接N1（连接前可以PING下N1的IP，测试是否能联通）
        • CMD或者终端输入：adb connect %IP%(此处替换为N1的IP)
      • 连接成功后，让N1进入fastboot模式
        • CMD或者终端输入：adb shell reboot fastboot后重启

  4. 提示Recovery则降级成功，断电重启N1即可完成降级

二、制作U盘镜像

  1. 电脑插入U盘

  2. 通过usb烧录软件，比如：balenaEtcher、USB Image Tool

  3. 选择固件镜像文件（各种固件镜像），写入U盘

    OS X用户的话，需要双系统或者虚拟机实现

  4. 烧录完成，先不要插入U盘

三、写入系统

若是系统没有刷入EMMC，则需要一直插着U盘，才能进入系统。（相当于，系统是刷进U盘的，没有刷入到N1中）

0. 原理

安卓系统的标准启动过程如下：

  • bootloader：斐讯针对该盒子硬件实现的类似BIOS的东西，它负责硬件的初始化，最终引导操作系统。
  • boot分区：放着安卓操作系统的内核文件，由bootloader唤起，也就实现了进入操作系统的效果。
  • recovery分区：类似于WinPE的东西，如果boot分区挂了，则可以通过Recovery方式进入一个临时的操作系统，然后恢复boot分区（类似于ghost做系统）

但是，bootloader是斐讯针对盒子定制的，它设置了一些限制

  • 只能使用他家的安卓系统
  • 不支持U盘引导操作系统

所以，需要Webpad大神的方法，修改斐讯的bootloader

参考：斐讯N1 – 完美刷机Armbian教程

N1刷电视盒子&主旁路由&小钢炮教程

悟空的 救砖和刷完XX系统再像刷其他系统方法 断电，拆机，短接，插电源，刷固件工具，重新刷固件

1. 刷入OpenWRT系统

参考：悟空教程

  1. 前面降级完，并且U盘已烧录进固件镜像，插上

  2. 电脑（网络）连接N1盒子，在同一个局域网

    • CMD或者终端输入：adb connect %IP%(此处替换为N1的IP)
      • CMD或者终端输入：adb shell reboot update后会自动重启

  3. 重启后，若出现4只企鹅，则OperWRT启动正常

若重启后出现机器人倒地，请多换几个U盘进行尝试

  1. 系统已经刷入，现在用网线把N1和电脑直连，输入IP测试是否能登陆上系统

    • 192.168.1.1
    • 账号：root，密码：password

一些问题

Q1：一些版本没有SSR，命令开启SSR

A：需要SSH登陆到盒子，通过命令开启SSR

  • echo 0xDEADBEEF > /etc/config/google_fu_mode

Q2：换皮肤主题

A：网上下载好皮肤ipk文件，通过ftp传到盒子里，然后ssh输入命令

  • mount_root #获取权限
  • opkg install 皮肤名字.ipk

OpenWRT旁路由设置

  1. N1连接电脑

    • N1与电脑，网线直连

  2. 确保电脑与N1在同一网段（电脑能自动获取到IP）

  3. 浏览器通过IP进入Openwrt

  4. 更改N1的网关地址（因为是旁路由，错开主路由IP）

    • 网络-接口-LAN-IPv4地址- 写IP
      • 若你的主路由IP是192.168.10.1，就写192.168.10.N(2-254都可以)

  5. 添加自定义DNS

    • 网络-接口-LAN- 自定义DNS服务器 - DNS
      • 114.114.114.114
      • 8.8.8.8
      • 添什么DNS，还是得看你所在的区域DNS速度

  6. 禁用DHCP的IPv6

    • 网络-接口- DHCP服务器 - IPv6设置
      • 禁用 路由器通告服务
      • 禁用 DHCPv6服务

  7. 保存

  8. 让N1连上网络，（使N1能上网，电脑也能通过N1上网）

    • 把N1盒子网线连接上主路由

N1单臂做主路由设置

  1. 连接方式
    • 光猫为直连模式，取消自己拨号，让N1拨号
    • 光猫、N1、电脑都连接交换机
  2. N1的Openwrt系统中，添加接口—WAN—dhcp–账号密码
  3. 其他IP网段设置好
  4. 总结：相当于，把交换机当成N1的接口~

2.刷入Armbian 系统 （相当于一个小型Linux服务器）

Armbian：轻量级的Debian系统和为ARM开发板专门发行并重新编译的Debian系统(Ubuntu派生自Debian)。

dtb：armbian内核为了能够和这些外设正常工作，就要求提供一种叫做dtb的描述文件，就是针对各个硬件的驱动程序。

- dtb文件需要随着内核编译，所以不同armbian内核版本必须使用配套的dtb文件
- 斐讯N1的dtb文件已经被armbian收录到官方源码库里，但是负载很高，有网友编译了一个fix过的dtb文件

参考：斐讯N1探索手记#1 – 降级并刷入armbian系统

斐讯N1 – 完美刷机Armbian教程 讲解的非常清楚！原理！还有一些问题解决方法

  1. 选择一个Armbian 系统版本（具体选哪个，就选5.77，这是目前恩山论坛最新也是最广泛肯定的版本）
    • Armbian5.44 网络稳定，可刷入eMMC，不支持bbr （内核版本：3.14.29）
    • Armbian5.77 可刷入eMMC，替换dtb后稳定，支持bbr（内核版本：5.0.2）
  2. 修改刷入U盘的Armbian镜像文件的dtb引导文件（有的需要，有的不需要）
    • 具体的参考 Xiangsm大神的5.77dtb文件
    • 或者 斐讯N1 – 完美刷机Armbian教程
  3. 网络连接盒子，设置让盒子U盘启动（先不插U盘）
    • adb connect %IP%(此处替换为N1的IP) （先查看N1盒子的新IP地址）
    • adb shell reboot update后会自动重启
  4. 进入Armbian系统，步骤如下
    • 千万不要在安卓系统开机的情况下插入U盘，否则U盘中的文件权限会被安卓系统篡改！
    • 先让N1断电
    • 插上U盘到靠近HDMI的USB口
    • 然后通电即可进入armbian系统
  5. ssh客户端软件，ssh连接盒子，让你能通过电脑输入命令到N1盒子中
    • IP: 刚刚的重启后，IP地址会发送变化，想办法查看盒子的IP地址
      • 方法1：通过的扫描IP端口软件 ScanPort，扫描网段，和22端口。即可找到盒子的IP
      • 方法2：路由器中查看
    • 用户名：root
    • armbian默认密码：1234
    • ls -l /确认一下目录权限没有被安卓篡改，都是root用户即可
  6. 通过命令写入镜像文件
    • 其他版本Armbian系统，请参考：斐讯N1探索手记#1 – 降级并刷入armbian系统
    • 输入：nand-sata-install (它可以完成所有刷写emmc所需的事情)
  7. 完成后halt关机拔掉U盘，所有刷机工作就结束了

非必要操作：

  1. 配置时区

    • 执行armbian-config，选择personal，然后配置Timezone为chongqing即可

  2. 换apt源

    • 执行armbian-config，选择personal，选择Mirror中清华大学的那个即可

  3. 连接WIFI

    • 执行armbian-config，选择network -> wlan0，选择wifi信号输入账号密码即可。

有以下问题，请参考：斐讯N1 – 完美刷机Armbian教程

Q1：开机后，偶尔无法联通外网

Q2：armbian的有线网卡每次重启后MAC地址都会变

Q3：ssh终端无法显示中文，也无法输入中文

刷armbian流程逻辑

  • 把armbian刷到U盘上，然后通过U盘引导进入armbian系统。
  • U盘进入系统后，我们执行一个命令就可以把armbian系统的boot分区以及rootfs等等都copy到emmc上。
  • 一旦完成emmc刷写，此后bootloader就会直接读emmc中的boot分区，直接引导进入armbian。

这里刷完Armbian系统后，可玩性就很高了，

因为Armbian可以装一些Linux的玩意儿，Docker~

如果你的openWRT想装进emmc，那么就先装Armbian，再把openWRT挂载上就可以了

3.刷入CoreELEC系统 （Kodi）

高清蓝光原盘解码无卡顿

官方：Github，官方中没有对应N1的系统，系统重新替换成N1对应的脚本，用下面恩山论坛的

恩山固件地址：N1适配的CoreELEC-9.2最终版本发布

视频教程：悟空Youtube

其实没必要把CoreELEC系统 刷入emmc，就专门拿一个U盘当系统盘，插入即可使用。

N1盒子本身就刷个安卓系统，如果想切换，插入U盘重启即可

  1. 前面先降级，并且U盘已经刷入固件
  2. 插入U盘，重启，即可进入CoreELEC系统
  3. 若想写入eMMC，则进行下面的步骤
  4. 先开启SSH
    • 设置-最底部的CoreELEC图标–服务–开启SSH
    • 可以同时设置密码，也可以不设置
    • 不设置密码的话（默认的是 账号root 密码coreelec）
  5. SSH连接盒子
    • 登录后执行installtointernal，按屏幕提示操作完成
