JAVA教程一之说明和安装

书名：疯狂Java讲义（第4版）

缩写
环境

  1. JRE 运行环境（Java Runtime Environment），包含JVM，只是运行java程序的话，就只需要这个
  2. JDK 开发环境（JAVA SE Development Kit，JAVA标准开发包），包含JRE，开发必备
  3. JVM JAVA虚拟机
  4. JIT 即时编译编译器

技术

  1. J2ME（改成Java ME） 用于控制移动设备和信息家电
  2. J2SE 整个JAVA技术的核心和基础
  3. J2EE（改成JAVA EE） 提供了企业应用开发相关的解决方案

一、JAVA技术分类

  1. Java SE 核心和基础
    • JDK 开发环境，包含JRE与JVM
    • Java EE SDK
  2. Java EE 企业应用开发解决方案
    • JDK
    • Java EE SDK
  3. Java ME 移动设备和家电

二、JAVA的运行机制

1、高级语言

JAVA即是编译型语言，又是解释型语言。先编译，后解释

2、何为编译型，何为解释型

编译型

所有的代码都会由“编译器”来编译，编译成适合对应平台硬件的机器码，并包装成该平台可执行的程序格式。编译好后的程序，可以脱离开发环境而执行

  • 优点：运行效率较高，
  • 缺点：一般无法跨平台
  • 还有哪些语言：C、C++、Objective-C、Swift、Kotlin

解释型

所有代码都会由专门的“解释器”，逐行解释成特定平台的机器码并且会立即执行。

  • 优点：跨平台（每个平台都有特定的解释器），方便移植
  • 缺点：每次都需要编译一次，效率低
  • 还有哪些语言：JavaScript、Ruby、Python

3、JAVA的运行机制

JAVA的源代码文件，先会由编译器编译成 *.class文件（JAVA解释器才能打开），然后由JAVA虚拟机负责解释。

两个步骤：先编译，后执行

JAVA虚拟机（简称：JVM），在不同平台上都不一样，但接口是一样的。

打个不恰当的比方，JAVA虚拟机的接口就像数据线，一头是对应各种平台的接口（比如苹果接口，安卓接口，Type-C接口等等），而另一头则是通用的USB接口（*.class文件）。

若某个平台有对应的JVM，则可以运行JAVA代码。

就像一个转换器，一个翻译器

再比如，你与 外国人对话，你们之间放了一个可以翻译多国语言（不同平台）的翻译器。你深思熟虑的说完所有的话（编译好后的字节码文件），翻译器内部翻译解释，说给这个外国人听（特定的机器码），如果是不同国家的人，翻译器也能翻译。

三、JAVA安装

  1. 官网下载对应平台的 JAVA JDK
  2. 下载好后，傻瓜式下一步就好了
  3. 建议自定义安装目录，然后添加到环境变量
  1. 添加系统变量
    • 变量名：JAVA_HOME
    • 变量值：刚刚自定义的安装目录，比如 C:\Program Files (x86)\Java\jdk1.8.0
  2. path 中添加：%JAVA_HOME%\bin;%JAVA_HOME%\jre\bin;
    • %JAVA_HOME%\bin
    • %JAVA_HOME%\jre\bin
  1. CMD或终端中测试下,如果有版本信息，说明安装成功

Bat设置环境变量

  1. 修改第三行，你自己的java路径
  2. 检查你的ClassPath是否有设置过其他软件，没有则随意使用
    • 其实可以追加，不用做覆盖（与path设置方法一样）
    • 但是不一定会有classpath，需要另作判断，懒，这里就不优化了。。。

安装后，目录中文件夹对应解释

  • bin：JDK的各种工具命令（编译器和工具）
  • conf：JDK的相关配置文件
  • include：一些平台特定的头文件（用于编译本地方法的文件）
  • jmods：JDK的各种模块
  • legal：JDK各模块的授权文档
  • lib：JDK工具的一些补充JAR包（类库文件）

2 终止

任务管理器方法

  1. 打开任务管理器
  2. 一般是 Windows命令处理程序，如果有多个，需要分辨，继续看
  3. 在上方标题栏 - 右键 - 选择（增加），命令行
  4. 排列下，找到java命令的进程，终止即可
    • 如果找不到，可能被缩放了，需要展开查看（在名称处，有个箭头，点下就出来了)

端口号方法

终止场景：想要终止运行，发现找不到cmd窗口，只能强行结束进程。

解决办法：找到端口号对应的进程，然后kill

  1. 首先用netstat -ano | find “端口号”查出进程号pid

  2. taskkill /f /pid 端口号 杀死pid
