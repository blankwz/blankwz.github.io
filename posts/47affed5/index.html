自制L大固件方法

参考

  • L大github
  • OpenWRT固件编译
  • OpenWrt 编译步骤与命令详解
  • eSir的Youtube
  • 利用GitHub ACTIONS 在线云编译L大的LEDE固件，小白无脑输出，省去本地搭建环境的麻烦
  • 谷歌云NanoPi R1S-H5固件编译安装全过程_Youtube
  • 编译Lean的Openwrt固件全攻略

不要用 root 用户 git 和编译！！！
国内用户编译前最好准备好梯子
默认登陆IP 192.168.1.1, 密码 password

1.安装Ubuntu

用虚拟机安装，或者Win10的Microsoft Store安装Ubuntu（网上随便搜一下教程很多，很简单）

  • 创建新用户，不能用root
    如果是Win10的Microsoft Store安装Ubuntu，需更改注册表
  • HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Lxss{GUID}\ 把Flags的7改成5

    2.翻墙环境

  • 全局模式，
  • 或者ubuntu网站加入黑白名单，强制代理…(dl包太多了，还是不要了。直接全局吧)

    3.执行代码

  • 命令行输入 sudo apt-get update
  • 然后输入 sudo apt-get -y install build-essential asciidoc binutils bzip2 gawk gettext git libncurses5-dev libz-dev patch python3.5 unzip zlib1g-dev lib32gcc1 libc6-dev-i386 subversion flex uglifyjs git-core gcc-multilib p7zip p7zip-full msmtp libssl-dev texinfo libglib2.0-dev xmlto qemu-utils upx libelf-dev autoconf automake libtool autopoint device-tree-compiler g++-multilib antlr3 gperf

    出现问题：因为自己C盘空间不足问题，导致解压包空间不够，导致出错卡住，然后报错： Unmet dependencies. Try ‘apt –fix-broken install’ with no packages (or specify a solution).
    解决办法 : sudo apt –fix-broken install 执行修复命令 ,然后sudo apt-get update，然后sudo apt-get upgrade。重新执行更新和升级

  • 使用 git clone https://github.com/coolsnowwolf/lede 命令下载好源代码，然后 cd lede 进入目录
  • ./scripts/feeds update -a
  • ./scripts/feeds install -a
  • make menuconfig

    出现问题：Build dependency: Please install ncurses. (Missing libncurses.so or ncurses.h) ，解决办法sudo apt-get install libncurses5-dev ，然后make
    make后，进入功能选择界面。感觉界面不对。
    心慌，编译后还是不行咋办~心一横，算了，重来吧。
    把ubuntu重置（应用设置-重置）

4.功能选择界面

快捷键yushuoming

‘*’ 代表编入固件，‘M’ 表示编译成模块或者IPK包， ‘空’不编译

  • 选中：空格
  • 选择：回车
  • 退出：两下esc
步骤：

  • Target system：选择x86（我们是软路由）

  • Subtarget:x86_64 (不用改)

  • Target Profile:Generic (不用改)

  • Target Images:

    • squashfs:选中 ，最重要
    • 其他不重要

  • Global build settings：没什么要改的

  • Base System：

    • dnsmasq-full:不要和dnsmasq选重复了，会冲突
    • build with dhcpv6：你没有需求就不用选
    • use fullconenat:据说会提升游戏速度

  • LuCI：所有系统组件都在这
    Applications： 精华

    - adbyby-plus：去广告
- aria2：下载工具
- ddns：远程访问需求，内网访问
- frpc：内网穿透
- hd-idle：连接硬盘共享
- sfe：转发加速
- shadowsocks
- v2ray
- kcptun
- ssr
- ssr and tunnel
- ssr pro
- ssr python
- v2ray pro

  • Network:

    • download Manager:下载工具选择
      • ariang:推荐

  • Utilites

    • Editors：
      • nano：编辑修改文件（比vi好用）

完成！SAVE！EXIT

5.编译

  • 预下载编译所需的软件包：make download -j8 V=s
    • 下载必要的包，如有压缩包错误，请进dl目录删除对应的包，回根目录执行 make clean 后，重新执行此命令。也可以直接将dl文件夹删除后再重新执行此命令

也可以网上找一些下载好的dl包，然后导进来(lede/dl文件夹，最后注意权限)，我是开了全局，几分钟下好~
没开全局前。。。下载了1个多小时没下好，放弃了。实在忍不住，就全局下了~
至于为什么开始没全局….因为在看电影，舍不得那点流量~~

  • 第一次编译：make -j1 V=s

    -j1：使用单线程编译。新手推荐单线程编译，一是因为玄学问题可能成功率高，二是方便查看错误日志，多线程的错误日志是交织在一起的，不方便排错。
    V=s：输出详细日志，用于编译失败时找出错误。

  • 第二次编译后：make V=99 -j2

如果编译错误，或者权限问题

遇到的错误（Windows10的store安装Ubuntu必会出现的问题）

错误：find: The relative path 'Files' is included in the PATH environment variable, which is insecure in combination with the -execdir action of find. Please remove that entry from $PATH
解决方法：更改注册表：HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Lxss{GUID}\ 把Flags的7改成5

6.找到编译好后的固件

  • Win10安装的Ubuntu位置：

    C:\Users\（你Win10系统的用户名）\AppData\Local\Packages\CanonicalGroupLimited.UbuntuonWindows（开头文件夹）\LocalState\rootfs\home\（你Linux系统的用户名）

  • 编译后的固件文件在 /lede/bin/targets 的 squashfs-sysupgrade.bin结尾文件
    所以，固件位置位于：

    C:\Users\（你Win10系统的用户名）\AppData\Local\Packages\CanonicalGroupLimited.UbuntuonWindows（开头文件夹）\LocalState\rootfs\home\（你Linux系统的用户名）\lede\bin
    (地址可能会不一样，根据系统用户名，和其他因素决定)
