使用RSS订阅信息

RSS 的全称有三种说法，RDF Site Summary（资源描述框架站点摘要）、Rich Site Summary（网站内容摘要）和 Really Simple Syndication（简易资讯聚合）

对于多数人，你这样理解就够了：

RSS是一个能让你在一个地方订阅各种感兴趣网站的工具,且让用户在不打开网站的情况下快速地获取到自己订阅的信息

为什么要用RSS

  1. 拒绝推荐算法
  2. 只获取自己想要的信息，避免无效信息；
  3. 提高效率，节省大量浏览时间；
  4. 真正做到内容聚合

一、如何使用

  • 部分网站有自己的RSS源，直接复制地址，粘贴到RSS工具中即可
    • 通常在顶部菜单、右侧菜单、底部菜单等地方会有RSS图标。
    • 如果没有，可以尝试在网站地址后面加上/rss或者/feed，有时会出现在二级域名里
  • 没有RSS源，就通过第三方工具，生成订阅源，比如下面的 RSSHub

一些源规则

  • Reddit：在当前链接后面加入.rss

    • 比如 https://www.reddit.com/r/nba/top/改成 https://www.reddit.com/r/nba/top/.rss

  • V2EX

    • 周报：http://vdaily.iu.vc/old-weekly.xml

  • Github：

    • 用户动态: 地址 + .atom，比如 https://github.com/runningcheese.atom
    • 仓库releases: 地址 + .atom，比如 https://github.com/soimort/you-get/releases.atom
    • 仓库commits: 地址 + .atom，比如 https://github.com/runningcheese/RunningCheese-Firefox/commits/master.atom

三、Reeder使用

Reeder 3 就是一款 iOS 和 macOS 系统下非常好用的软件（在 A 君眼里，把它称之为 Google Reader 之后最好用的 RSS 阅读器也不为过），它足够简单、美观

在 Reeder 3 中如何导入 RSS 源和账号

要在 RSS 阅读器中进行阅读，首先你必须要把 RSS 源加入到阅读器中，就像微信公众号一样，你只有关注了它，才能收到它推送的文章。

通过其它 RSS 阅读器导入

首先，你需要创建一个账号。这里推荐创建 Feedly 账号，因为 Feedly 是目前最好用的 RSS 服务网站。

打开 Reeder 3 之后，点击顶部的「Tap to add an account」- 第三个服务「Feedly」- 「Continue with feedly」，没有账号则点击最底部的「New user? Sign up」注册新账号，有账号则直接输入账号密码登录即可。

五、自建RSS订阅服务平台

FreshRss

  • Github

  • 官网地址：https://freshrss.org

    演示地址：https://demo.freshrss.org

  • 安装

    • 无宝塔面板，安装方法
    • Docker安装方法
    • 还有下面的利用宝塔的简单安装

简单安装

  1. 宝塔后台先添加新站点和数据库

    • 写个二级域名，再域名绑定IP

  2. 环境要求：Nginx PHP >= 5.3 MySQL >= 5.5

    • 需要Fileinfo组建支持（软件商店 - php设置 - 安装扩展 - 安装Fileinfo）

  3. 下载 FreshRSS 源码，上传至网站根目录，并解压所有文件至根目录

  4. 打开你的域名就可以进入 FreshRSS 的安装界面了

  5. 跟着提示进行设置就行了，数据库名和用户密码在宝塔后台数据库都能看到，照着填写就OK了

参考：参考1 和 参考2

安装错误

openlog 报错：Warning: openlog() has been disabled for security reasons in /www/wwwroot/rss.mianao.info/lib/lib_rss.php on line 15

解决办法：

到 /www/server/php/70/etc/php.ini 打开 php.ini 文件到 299 行（附近），删除 disable_functions = 后面的 openlog, 就好了

然后重启 PHP（可在宝塔面板重启，也可以用命令）

更新Feed

  1. 下载扩展 Auto-Refresh
  2. 将解压后的 xExtension-AutoRefresh 文件夹放到/www/wwwroot/你的站点/extensions 目录下
  3. 到 FreshRSS 设置-> 扩展启用就好了
  4. 默认是 1 分钟刷新一次，想修改可以到 xExtension-AutoRefresh/static/script.js 打开这个 js 文件，里面是 60000，单位应该是 ms

手机访问

  1. FreshRSS 设置->用户账户->API 管理，设置API密码。
  2. 管理 - 认证 - 勾选 允许API访问（用于手机应用）
  3. 打开Reeder，添加账号 - 选择FreshRSS
    • Server：https://域名/p/api/
    • user：FreshRSS 的登录名
    • 密码：上面设置的API密码

注意

  1. 在订阅管理中，Tag和 Category 不能相同，否则无法创建

六、其他玩法

1、监视网页内任意内容的变化。
有一款叫 Distill Web Monitor 的拓展 （支持 Firefox / Chrome），可以让你监视网页某处内容的变化，并在第一时间通知你。

使用的场景非常多，比如监控某商城网站商品是否有货是否有降价，某网站上的房价涨跌提醒，某个页面是否有内容更新，某视频网站Po主发布新视频的提醒，再比如你是做运营的，想要监视竞争对手产品的动态，这款拓展都能做到。更加强大的地方在于它可以自定义提醒的条件，比如价格变化超过10%才提醒你。

2，指定新闻内容动态提醒。

Google Alerts 可以让用户指定监控的关键词，Google 会在第一时间内向用户推送新内容更新提醒，支持邮件提醒，也支持RSS提醒，非常方便，用户可以用这个功能来跟踪一些新闻报道，业界动态，获取最新的国际事件等等。

Google Alerts的最大作用就是：让用户能迅速而方便地获得其所关注的信息。Google Alerts能将有价值的信息主动推给用户，大大减少了用户获取信息的时间。实现了“不上网而知天下事”。百度也有类似产品。

3，RSS 配合 IFTTT

IFTTT是“if this then that”的缩写，可以让你的网络行为能够引发连锁反应。以RSS为例，比如在你的RSS订阅源里出现了有关于”iPhone8”的信息，联接上IFTTT后就可以在手机上弹窗通知我们了。
